Using EMBED in multiple frames

Synopsis

Sometimes sonified documents that work reliably on their own cease to work reliably when placed together inside a frameset.

Symptoms

There appears to be a JavaScript plug-in reflection problem when having plug-ins in multiple frames of the same frameset. Sometimes plug-in handles, as defined in the NAME attribute, fail and it is impossible to speak to the plug-in using LiveConnect. When querying Navigator to find the length of the embeds[] array, a value of 2 is sometimes returned for a document which most absolutely only has one instantiated plug-in. In the cases where the reflection fails, the symbolic plug-in handle (as defined in the NAME attribute) returns a value of null. In cases where the reflection succeeds, the result returned is something like "Rmf@400025b8".

Solution

We've resorted to always only having music content in one frame. This imposes certain unpleasant restrictions on authoring, however, this has proved the only sure way of authoring reliable sonified Web sites that use framesets.
