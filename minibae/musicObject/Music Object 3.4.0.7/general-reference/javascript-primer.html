Useful JavaScript events for sonifying Web pages

JavaScript supports various events. These events enable you to script a response to a user's interaction with an HTML element on a Web page. The code that you write to respond to an event is termed an event handler. Event handlers can be used most effectively in making a user's interaction with a page trigger UI sounds or changes within an underlying musical score.

The events most commonly used in sonifying a Web page are outlined in the table below. Beatnik Inc. is not reponsible for the maintenance and extension of the JavaScript language and so you should view the table as merely a guide, and should always refer to Netscape's on-line JavaScript documentation as your primary JavaScript programming reference.

ONCLICK Occurs when the user clicks a form element or link. ONCLICK is an action event and is very appropriate for use in UI sonification. Other events such as ONLOAD, ONCHANGE, ONBLUR, ONSELECT, etc., are less directly connected to immediate user actions. buttons

ONMOUSEOVER Occurs when the user moves the mouse pointer over a link. This is one of the more useful events to use in sonification as it allows one to sonify a user's movement over a Web page. Done in a tasteful manner, ONMOUSEOVER events can be used to script fun and engaging pages which give immediate gratification to the user. links
ONMOUSEOUT Occurs when the user moves the mouse pointer out of an area (client-side image map) or link. Paired with the ONMOUSEOVER event, you will find this event to be one of the most frequently used in scripting sonified Web pages. links
