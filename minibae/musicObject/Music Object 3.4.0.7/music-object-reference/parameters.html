 BankNumber0to2

 
 CATEGORY   Parameter 
 Type   integer 
 Values   0 = GM Bank 
 1 = Special Bank 
 2 = User Bank 
 Constrained   no 
 

The Beatnik soundbank contains 2 banks, each with a full 128 instrument sounds and a percussion set. There is also an additional bank, called the user bank, which is available for composers to define their own custom instruments to include in their RMF compositions.

The banks are as follows...

Bank# Bank Name Description
0 GM bank A General MIDI soundbank modeled to be closely compatible with established GM soundbanks such as the arguably industry-standard Roland SoundCanvas GM soundset. Content which has been authored for a GM sound module should be flattered by the Beatnik GM bank.
1 Special bank The Special bank contains very cool textures, some of which stand in as alternatives to their less adventurous GM counterparts in bank 0. Often a program number in the GM bank will have an instrument which is mirrored at the same location in the Special bank, except that the Special bank version will have a refreshingly different slant on it. This is convenient and makes it quite easy to change the feel of a composition merely by changing a bank select or two in the sequence, while still having the mix make musical sense.
2 User bank The User bank is where composers of RMF content can place their custom instruments. Their are no instruments in the User bank of the Beatnik soundbank and so this space is open for RMF composers. The custom instruments in the User bank are usually included in the RMF file which is exported from the Beatnik Editor.

For the list of instrument sounds contained in the various banks, consult the interactive instrument list.

  • Methods using this parameter: noteOn , playNote , playRandomNote , setProgram.

 FadeTimeMsINT

 
 CATEGORY   Parameter 
 Type   integer 
 Maximum   no upper limit 
 Minimum   0 
 Constrained   no 
 

FadeTimeMsINT is an integer value indicating the amount of time over which a volume fade is to occur. It is measured in units of a millisecond (1/1000th of a second). Methods that take FadeTimeMsINT as a parameter can also typically take the FadeBOOL parameter in place of it.

  • Methods using this parameter: fadeTo , fadeVolume , stop , pause.

 FadeBOOL

 
 CATEGORY   Parameter 
 Type   boolean 
 Values   true 
 false 
 

FadeBOOL is a boolean parameter indicating whether a volume fade should occur or not. A value of false indicates that no volume fade should occur, and a value of true indicates that a volume fade of 1000 milliseconds (1 second) should occur. Methods that take FadeBOOL as a parameter can also typically take the FadeTimeMsINT parameter in place of it. This allows more precise control over volume fades.

  • Methods using this parameter: fadeTo , fadeVolume , stop , pause.

 FileUrlSTR

 
 CATEGORY   Parameter 
 Type   string 
 

The FileUrlSTR parameter is a string specifying a media resource that may be located on the Internet or on the local system. The syntax for this parameter is exactly the same as the normal syntax used in referring to HTML page media such as images, page links, page backgrounds, etc.

Using a relative URL syntax allows one to author content that will run locally without the user being connected to the Internet - vital for creating portable content that can be run off a removable storage medium such as a ZIP disk, external SCSI hard drive, CD-ROM, etc.

In addition to the established set of protocols - such as "http:", "ftp:", "file:", and others - the Beatnik Player also supports a non-standard "groovoid:" protocol, which allows authors to specify one of the player's built-in Groovoids when using the play method.

  • Methods using this parameter: play.

 Function

 
 CATEGORY   Parameter 
 Type   function 
 

The Function parameter specifies a user-defined function that will be used to handle a callback from the Beatnik Player. When the function is specified in a method such as onPlay, the function must already be defined in the code, otherwise an undefined function value will be passed to the callback method and the code will not behave correctly. The function should, therefore, be defined ahead in the code before any reference to it is made.

  • Methods using this parameter: addCallbackHandler , addMetaEventHandler , addOnLoad , addOnMetaEvent , addOnPause , addOnPlay , addOnReady , addOnStop, fadeTo , fadeVolume , onLoad , onMetaEvent , onPause , onPlay , onReady , onStop , removeCallbackHandlers , removeMetaEventHandlers , removeOnLoad , removeOnMetaEvent , removeOnPause , removeOnPlay , removeOnReady , removeOnStop.

 LoopCountINT

 
 CATEGORY   Parameter 
 Type   integer 
 Maximum   no upper limit 
 Minimum   0 
 Constrained   no 
 

LoopCountINT is an integer value representing the number of times a file is to play through. For example, a value of 1 means that a file will play through only once. This is effectively the same as using false as a value for the LoopingBOOL parameter - playing through only once is in essence the same as not looping. However, using a value higher than 1 will result in the file looping, with as many repititions as is specified in the LoopCountINT parameter. One special case is using a value of 0 for this parameter, which will result in the file looping indefinitely. This is effectively the same as using true as a value for the LoopingBOOL parameter. Methods that can take the LoopCountINT parameter can also typically take the LoopingBOOL parameter in its place.

  • Methods using this parameter: setLoop , play.
  • Methods returning this parameter: getLoop.

 LoopingBOOL

 
 CATEGORY   Parameter 
 Type   boolean 
 Values   true 
 false 
 

LoopingBOOL is a boolean value specifying whether or not a file is to be looped by the player. A value of false will result in the file playing only once and then stopping, and a value of true will result in the file looping indefinitely (until it is stopped by the user or a Music Object method such as stop). Methods that can take the LoopingBOOL parameter can also typically take the LoopCountINT parameter in its place.

  • Methods using this parameter: setLoop , play.
  • Methods returning this parameter: getLoop.

 MidiChannel0to16

 
 CATEGORY   Parameter 
 Type   integer 
 Maximum   16 
 Minimum   0 
 Constrained   no 
 

MIDI channel numbers range from 1 to 16 (with MIDI channel 10 being the percussion channel), but in many cases MIDI channel number 0 acts as a "magical" channel number which indicates that the method should apply to all channels, 1 through 16.

eg. You could set the volume of all channels to maximum by using a statement such as myMusicObject.setController (0,7,127), where the MIDI channel of 0 indicates that controller 7 of each channel should be set to 127. The magical MIDI channel number 0 does not apply to "get" type methods which return a value.
  • Methods using this parameter: setController , setChannelMute , setChannelSolo , setTransposable , setProgram.

 NoteNameSTR

 
 CATEGORY   Parameter 
 Type   string 
 Values  Consult the Appendix Note Names and Numbers
 

NoteNameSTR is a string used to represent a valid MIDI note by its symbolic name - a name that can be understood by musicians. The syntax for this string is...

note (C,D,E,F,G,A,B) | optional sharp (#) or flat (b) specifier | octave number

For instance, middle C (C in the 4th octave) is represented by the string value C4. And F sharp in the 6th octave is represented by the string value F#6. And as a final example, D flat in the 3rd octave is represented by the string value Db3. Consult the Appendix Note Names and Numbers for a full listing of MIDI note numbers and their corresponding names. Methods that take NoteNameSTR as a parameter can also take the NoteNumber0to127 parameter in its place.

  • Methods using this parameter: noteOff , noteOn , playNote , playRandomNote.
  • Static methods using this parameter: Music.getNoteNumber.
  • Static methods returning this parameter: Music.getNoteName.

  • Reverb types 1 through 7 are old reverbs which do not offer variable send. Reverb types 8 and above are newer reverbs supported from Beatnik Player 2.0 onwards, which do offer variable send.

 TrackNumber0to64

 Maximum   64 
 Minimum   0

Tracks inside an RMF or MIDI file are numbered starting at 1. The Beatnik Player can handle up to a maximum of 64 tracks. In many cases track number 0 acts as a "magical" track number which indicates that the method should apply to all tracks in the RMF or MIDI composition.

eg. You could mute all tracks of a composition by using a statement such as myMusicObject.setTrackMute (0,true), where the track number of 0 indicates that the setTrackMute method should set the mute state of all tracks in the composition. The magical track number 0 does not apply to "get" type methods which return a value.
  • Methods using this parameter: setTrackMute , setTrackSolo.

 Velocity0to127

 
 CATEGORY   Parameter 
 Type   integer 
 Maximum   127 
 Minimum   0 
 Constrained   yes 
 

Velocity0to127 is a MIDI concept and represents the strength with which a musical note is triggered. Depending on the instrument's definition, velocity quite often translates to the overall volume of the note triggered, but this is not necessarily the case. Instruments are sometimes defined so that filters are controlled by velocity as well, resulting in significantly different sound "colors" at different velocity values.

  • Methods using this parameter: noteOn , playNote , playRandomNote.

 Volume0to100

 
 CATEGORY   Parameter 
 Type   integer 
 Maximum   100 
 Minimum   0 
 Constrained   yes 
 

Volume level is a measure a player instance's overall volume and is measured in a range of 0 to 100, where a value of 0 represents silence and a value of 100 represents maximum volume. Volume level can be set using the setVolume method and can be obtained using the getVolume method. Other methods that can affect the current volume level of a player instance are fadeTo and fadeVolume. Volume level is not global and setting the volume level of a player instance does not affect the volume levels of other instances. Volume level is also not to be confused with MIDI controller 7 which can set the volumes of individual MIDI channels in an RMF or MIDI file.

  • Methods using this parameter: fadeTo , fadeVolume , setVolume.
  • Methods returning this parameter: getVolume.
