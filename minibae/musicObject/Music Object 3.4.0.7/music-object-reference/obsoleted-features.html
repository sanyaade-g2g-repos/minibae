Obsoleted Features

Many versions of the Music Object API have come and gone, and some significant changes in the API have occurred, leaving a good number of methods and properties either deprecated or completely unsupported. This section is provided to help developers in updating old sonification code to use the most current version of the Music Object API.

This section lists all methods and properties that have become unsupported, with explanations on how those methods and properties behaved and how newer methods and properties of the Music Object API can be used to achieve similar results. Wherever possible, explanations on how to update code are provided.

 stopAll

 
 CATEGORY   Instance Method 
 Status   no longer supported 
 Introduced at Version...   2.2.2 
 Obsoleted at Version...   3.1.0 
 Current Equivalent   Music.stopAll 
 

The stopAll instance method provided a way to stop playback for all player instances in a single operation.

    stopAll ( )

History
  • The stopAll instance method was first introduced in Music Object 2.2.2.

  • For the release of Music Object 3.1.0, it was decided that it was no longer appropriate for this method to be an instance method, since the method could affect multiple Music Object instances - not just the instance that the method was called on. Consequently, this instance method was supplanted by the Music.stopAll static method.

The Music.stopAll static method is not functionally equivalent to the obsoleted stopAll instance method. It was decided that it was no longer desirable to provide a method that could have the effect of stopping playback of all player instances - including player instances that were not part of the document that originated the call to the method - since this could result in the sonification for one Web site interfering with the sonification of a different Web site that is contained inside a different window. So, the Music.stopAll static method was implemented so that it would only stop the playback for all the player instances contained inside the document that originated the call to the method.

 musicObject

 
 CATEGORY   Global Function 
 Status   no longer supported 
 Introduced at Version...   1.1.0 
 Obsoleted at Version...   3.1.0 
 Current Equivalent   Music 
 

The musicObject global function was an object constructor that provided a way to create new Music Object instances.

    musicObject ( ObjectNameSTR )

History
  • The musicObject global function (object constructor) was first introduced in Music Object 1.1.0.

  • For the release of Music Object 3.0.0, it was decided to present a more elegant API to developers by rolling much of the functionality, that was previously available in the form of global functions and global variables, into a single object class. The object class name "Music" was chosen in order to bring the API in line with other objects that are built into the JavaScript language (such as the "Math", "Image", "Date" and other objects). Since the capitalization of the first letter now indicated that "Music" was an object constructor, it was considered redundant to append "Object" to its name. Consequently, the musicObject object constructor was replaced by the functionally equivalent Music object constructor. It was decided to still support the musicObject constructor to maintain backwards compatibility, giving content developers time in which to transition to using the new Music constructor.

  • With the release of Music Object 3.1.0, it was decided to no longer support the musicObject constructor. A backwards compatibility patch (Music Object Patch - Pre 3.1.0) was provided to allow content developers to use version 3.1.0 without having to update their code to accommodate this change in the API.

 mo_requiredMinVersion

 
 CATEGORY   Global Variable 
 Status   no longer supported 
 Introduced at Version...   2.6.9 
 Obsoleted at Version...   3.0.0 
 Current Equivalent   Music.requireMinPlayerVersion 
 

The mo_requiredMinVersion global variable was a string variable that specified the minimum player version required by a sonified page.

History
  • The mo_requiredMinVersion global variable was first introduced in Music Object 2.6.9.

  • In order to present a more elegant API, it was decided for the release of Music Object 3.0.0 that all global properties previously available as global variables prefixed with "mo_" should be supplanted by similarly named static properties of the Music object. Consequently, the mo_requiredMinVersion global variable was replaced by the functionally equivalent Music.requiredMinVersion static property.

  • Finally, with the release of Music Object 3.3.0, it was decided that the Music.requiredMinVersion static property should become a read-only property and that the newer Music.requireMinPlayerVersion static method should be used, instead, to request a minimum required version. This change was made to allow multiple separate code modules to independently request a minimum player version, such that the highest minimum player version that is requested by all code modules will be the version used to determine if the player installed on the user's system is adequate for the content as a whole. This approach allows separate code modules to co-exist and declare their player dependencies in a more cooperative manner. With this new approach, the read-only Music.requiredMinVersion static property serves the purpose of indicating the minimum required player version that results from all requests for a minimum version that go through the Music.requireMinPlayerVersion static method.

Code Modules

 Music Object Patch - Navigator 4.04 Bug

 
 CATEGORY   Code Modules 
 Status   no longer supported 
 Introduced at Version...   3.1.0 
 Obsoleted at Version...   3.4.0 
 

The Music Object Patch - Navigator 4.04 Bug code module was used to work around a specific bug in Netscape Navigator 4.04.

History
  • After the bug in Navigator 4.04 was discovered, a workaround was introduced and was built into Music Object 2.1.6.

  • The built-in workaround was subsequently dropped in Music Object 2.5.5, due to several releases of Navigator passing since Netscape fixed the bug and also due to the code size that the workaround added to the Music Object API.

  • In Music Object 3.1.0, due to concerns from Web content developers regarding compatibility of their sonified content with all versions of Navigator, this workaround was reinstated in the form of an optional code module "Music Object Patch - Navigator 4.04 Bug". This provided the best way at the time to offer content developers the option of supporting the old 4.04 version of Navigator, while at the same time not bloating the code size of the core Music Object API.

  • Finally, with the release of Music Object 3.4.0, Beatnik officially stopped supporting versions 4.0.x of the Netscape Navigator browser. As a result, the Music Object Patch - Navigator 4.04 Bug code module was obsoleted.
