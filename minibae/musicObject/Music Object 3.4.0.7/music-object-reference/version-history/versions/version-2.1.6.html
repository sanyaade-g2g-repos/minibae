What's New In This Version

Music Object Version 2.1.6 offers a workaround for a serious Navigator 4.04 issue that has caused existing content to break.

Navigator 4.04 Issue

If you have done or are planning to do any interactive sonification using Beatnik, you need to be aware of this issue. With the release of Netscape Navigator 4.04, it appears that a new bug / issue has been thrown at authors of Beatnik content. This means that some content you may have already authored is likely to not work in Navigator 4.04.

Fortunately, there is a remedy for this problem which is fairly painless and doesn't require additional authoring. Read on.

When It Occurs

This bug manifests itself in Beatnik sonified Web pages that exist inside a frameset. Due to the nature of the bug, it does not cause problems for single HTML pages that are not in a frameset. A single page that currently works fine in Navigator 4.04 will stop working once it is thrown inside a frameset.

Symptoms

Frameset pages authored for Beatnik, which otherwise work perfectly on earlier versions of Navigator, now produce JavaScript error messages to the tune of "myMusicObject is undefined" or "myMusicObject has no properties". After displaying these JavaScript error alerts, interactive content then does not work as designed. This is NOT a bug with the Beatnik Plug-in or the "music-object.js" file. It is a bug with Navigator 4.04.

Explanation

It appears that Navigator 4.04 is targeting all callbacks that originate from a plug-in - not to the frame in the frameset that contains the plug-in - but the top frameset document itself. This means that a callback is no longer executed in the appropriate context in which it was generated, leading in our case to error messages such as "myMusicObject is undefined" or "myMusicObject has no properties". This is a change in the behavior from all previous versions of Navigator.

Because all plug-in callbacks are now targeted to the top frame in a frameset, this problem does not affect non-frameset Web pages, since the HTML page then is the top frame in the window.

The Solution

Fortunately, at Beatnik Inc. we were able to find a workaround to make this issue invisible to authors. It has always been our goal to hide as much of the annoyance factor from Beatnik authors as possible. The workaround exists within the "music-object.js" JavaScript file. A process kicks into effect for Netscape Navigator browsers that determines the full frameset path from the top window to execute the callback function in the frame where the callback originated. So, even though the top frame is executing a function that would otherwise not be recognized in its context, it reaches back deep into the frameset hierarchy and executes the function in the appropriate context.

HOW TO UPDATE YOUR CONTENT

Despite the previous technical explanations, updating your content to avoid this Navigator 4.04 bug is quite easy.

  1. Update

    You will most likely be upgrading from Music Object version 1.x (if you are unsure which version you are currently using, consult Which Music Object Version Are You Using?). You will need to make some minor changes in your content in order to benefit from the workaround in version 2.1.6. Follow the simple instructions in Migrating Content From Music Object Version 1.x, and then continue with the remaining steps below.

  2. Test

    Now that you have updated your Web site(s) with the latest version of Music Object, check your Web site pages locally to see that you do not experience the bug described here. Be sure to use Netscape Navigator 4.04 since it is this browser that manifests the problem. If you use a version other than 4.04, you may not be able to tell if the change has been effective. Even if you didn't experience the bug before, do testing anyway to ensure that all your content still works with the new "music-object.js" file.

  3. Follow Up

    Once you have tested and are confident everything is peachy, make sure you have updated the online version of your Web site and watch closely for any e-mail reports from patrons to your Web site who may be browsing with different browser and OS configurations.

Recommendations

All authors should switch to the new "music-object.js" file immediately - even if you are not using framesets on your Web site. You may forget about this issue and later choose to place some single pages into a frameset and not understand why they are not working in Navigator 4.04.

  There may be other unrelated issues with Navigator 4.04 on the Mac or incompatibilities with early versions of the Beatnik Plug-in that may cause problems with Beatnik sonified content. If so, please bring these to our attention. This developer announcement addresses only this one very specific issue described above.
