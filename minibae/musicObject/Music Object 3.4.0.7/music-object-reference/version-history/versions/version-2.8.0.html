Music Object Version 2.8.0
RELEASE DATE: 1999-01-21

What's New In This Version

ISSUE WORKAROUND - Crash on Closing Internet Explorer
Using the most recent beta builds of the Beatnik ActiveX control, we had noticed fairly frequent crashes on exiting Internet Explorer. We traced this problem to an issue with Internet Explorer not freeing up all of the allocated Beatnik Player instances, while at the same time freeing up the Beatnik Player code that the instances depend upon. This results in a memory violation. We were able to work around the problem by modifying Music Object to proactively declare the player instances unneeded when the HTML document unloads.
ISSUE WORKAROUND - Music Persisting in Internet Explorer
The Internet Explorer issue that has already been documented, regarding music that persists even when the HTML document is unloaded, seems to have been related to the crash issue described above. The modification made to the Music Object code has fixed this issue as well.
