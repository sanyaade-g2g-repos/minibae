Music Object Version 2.6.9
RELEASE DATE: 1998-10-15

Many, many versions of Music Object have come and gone since the last version that was documented in this version history. Take this as a sign that Beatnik Inc. is improving its technology faster than it can document the improvements in its technology. Despite many revisions and many feature additions, the music-object.js file has grown by only around 2Kb since version 2.3.1 - quite an achievement considering how much has changed and how much is new!

What's New In This Version

NEW FEATURE - mo_magicEmbed
The new mo_magicEmbed global function is now offered as a way of embedding non-scriptable Beatnik Player instances on a page. This function offers all the Navigator/IE compatibility features of Music Object's similarly named magicEmbed instance method, but without the memory overhead of creating a Music Object instance. This function can be used in cases where scripting is not needed for a Beatnik Player.
NEW FEATURE - Compatibility Functions
Several new global functions have been added to enhance compatibility testing for different browser, operating system, and installed player version. These functions include the following...
  • mo_installBeatnik
  • mo_isPlayerCompatible
  • mo_hasMinVersion
  • mo_promptClose
  • mo_promptUser

In addition to these new functions, the compatibility testing logic has also been greatly improved. MusicObject has been linked directly to the online Beatnik Player Install Wizard, making it easier for user's who do not have the player installed to get setup quickly so they can proceed to experience your Beatnik-enhanced Web content.

NEW FEATURE - Compatibility Properties
As a companion to the new compatibility functions, the following compatibility properties (global variables) have also been added to further allow authors to customize Music Object's behavior when a user's configuration fails the author's requirements.
  • mo_ignoreJavaDisabled
  • mo_requiredMinVersion
  • mo_showCompatibilityPrompt
  • mo_silentIfInadequate
NEW FEATURE - New "play" Method Variations
The play method has been further enhanced to offer several new and convenient variations. Some of these variations make it possible to easily use the play method for playing Groovoids. New variations include...
  • play ( loop state , GroovoidNameSTR )
  • play ( 'auto' , GroovoidNameSTR )
  • play ( file URL )
  • play ( GroovoidNameSTR )
As a result of now being able to easily specify a Groovoid name where one would specify a file URL, the playGroovoid method has been deprecated. Content using the method should still work as normal, but authors should update their content to use the new variations of the play method that allow a Groovoid name to be specified in place of a file URL.
NEW FEATURE - Improved Debugging
MusicObject now extends its error catching capabilities to all frames of all framesets of a window that sources in the music-object.js file into any one of its frames. So, even errors that don't occur in the document that sources in music-object.js will be caught and reported through Music Object's debug mechanism, using the error reporting preferences you set.
OPTIMIZATION - Syntax
Various JavaScript syntactical changes were made to marginally reduce filesize.
OPTIMIZATION - General Improvements
Countless optimizations have been made since version 2.3.1 that have made it possible to add many features to Music Object while still keeping the code to roughly the same size.
BUG FIX - Debugging
The debug output function now checks to ensure that the "java" object exists, avoiding possible JavaScript errors in Internet Explorer.
