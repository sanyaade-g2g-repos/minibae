Music Object API Changes

GENERAL IMPROVEMENT: Support for Microsoft Internet Explorer on the MacOS

While Microsoft Internet Explorer on the MacOS has basic support for the Netscape Plug-in API, it does not support the LiveConnect system of the plug-in API (LiveConnect is a technology developed by Netscape to allow communication between Java code, scripting code in HTML pages, and plug-ins). Because of this missing feature it has not been possible to offer support for sonification in this browser, since the crux of Beatnik's sonification platform relies on the ability to "wire" HTML, DHTML, Flash movies, etc. to the powerful interactive audio features of the Beatnik Player.

So, it is with great pleasure that Beatnik can announce that it has developed its own proprietary system to work around this obstacle. In so doing, the Beatnik Player takes on the honor of being the very first scriptable plug-in for Microsoft Internet Explorer on the MacOS, and thereby further expands its broad range of platform support.

Users can now experience your sonified content in Microsoft Internet Explorer on the MacOS. A first!

 There are certain limitations and authoring considerations to be aware of when creating sonified content that is to be fully compatible with Internet Explorer on the Mac OS. For more information, see the section Issues for Internet Explorer on the MacOS.
GENERAL IMPROVEMENT: Improved Music.promptUser Static Method
The Music.promptUser static method has been improved so that the enter and escape keys can now be used in Microsoft Internet Explorer in order to trigger the ok and cancel actions for dialog boxes opened using this method. Previously, these keyboard shortcuts were only supported for Netscape browsers.
IMPROVED EXTENSIBILITY API: Improved Music.addKnownClient Static Method
The Music.addKnownClient static method of the Music Object Extensibility API has been improved to offer the additional CustomPropertiesOBJ parameter. This optional parameter lets you register additional information for a known client, which can then later be accessed and handled by your custom compatibility code.
IMPROVED EXTENSIONS: Improved Ducking Extension
The Ducking extension has been improved to include the convenient new Music.duckingAttackTime, Music.duckingSustainLevel, and Music.duckingReleaseTime static properties. These new properties make it easy for you to set your own logical default values for the AttackTimeMsINT, SustainLevel0to100, and ReleaseTimeMsINT parameters, so that these parameters can be omitted when setting up multiple ducking relationships. In addition to these new static properties, a bug that was found with the addDuckingMasters instance method was fixed for this version.
IMPROVED EXTENSIONS: Improved Song Info Extension
The Song Info extension has been improved to allow the newer "genre", "subgenre", "tempo description", and "original source" info field names to be specified in the SongInfoFieldsSTR parameter when calling the showSongInfo instance method.
IMPROVED EXTENSIONS: Instruments Extension Bug Fixed
A bug was found with the Music.getPercussionName and Music.getProgramName static methods and the getProgramName instance method. This bug has been fixed in maintenance release 1.1.1 of the Instruments extension.
GENERAL CHANGES: Netscape Navigator 4.0.x No Longer Supported
Because of a significantly dwindling installed base for 4.0.x versions of Netscape Navigator, and because of significant issues found specifically with versions 4.04 and 4.05, it was decided to discontinue support for 4.0.x versions of Navigator. Beatnik continues to support Navigator versions 4.5 and higher as well as the Netscape 6 browser. As a result of no longer supporting 4.0.x versions, the patch Music Object Patch - Navigator 4.04 Bug has been obsoleted.
Documentation Changes

NEW INFORMATION: Emulating Audio Device Contention
In order to help you in customizing your sonified content to best make use of the ADCM (Audio Device Contention Manager), a feature was added to Beatnik Player version 2.2 (Windows only) to allow you to coerce the player into thinking that the system either does or does not exhibit the audio device contention issue (regardless of whether it truly does or not). Among other things, this new feature allows you to test your ADCM settings on a Windows system that does not exhibit the audio device contention issue. This feature is described in full in the new section Emulating Audio Device Contention.
GENERAL IMPROVEMENT: New Company Branding
The Music Object API documentation has been updated to reflect the company's new corporate branding.
GENERAL IMPROVEMENT: More Printer-friendly

Several changes have been made to the layout of the documentation in order to help make it more printer-friendly. Among these improvements are...

  • No Underlined Links

    Links are no longer underlined, making the documentation much cleaner looking when printed. Sections containing large numbers of links now print out much less cluttered than before. Because links only apply to browsing the documentation, it is appropriate that links stand out less in print. Links can still be identified easily enough when browsing, since they are highlighted using a different color to the standard text. They also appear underlined when the mouse is moved over them (in browsers that support this feature).

  • No "EDIT" Buttons For Methodizer

    In previous versions of the documentation, an "EDIT" button would appear in certain method reference sections. This was to allow you to launch the Music Object Methodizer directly to those methods so that you could edit their parameters. This functionality still exists in the documentation, but there are no longer dedicated "EDIT" buttons. Instead, the method names for methods that can be edited in the Methodizer are linked to launch the Methodizer (for more information, see the section "Launching the Methodizer from Reference Sections"). Because there are no longer dedicated "EDIT" buttons, this convenient interactive feature that is only applicable to browsing the documentation is now hidden when the documentation is printed.

  • Improved Table Styles

    Styles for tables have been modified to improve how they appear in print. This includes making headings stronger and row and column boundary lines clearer.

GENERAL IMPROVEMENT: Improved Navigation
The listbox navigation system that is used in the Tutorials, Demos, Sample Code Examples, Version Histories, and various other sections throughout the documentation has been improved in order to make the system more intuitive. Before, the arrows on the left and right hand sides of the listbox would always be active, allowing you to "wrap around" from the start to the end (using the left arrow) and from the end to the start (using the left arrow). This behavior has been changed to present a clearer idea of when you are at the start or the end of a set of pages. When you are at the start, the left arrow is now "grayed out". And when you are at the end, the right arrow is "grayed out".
