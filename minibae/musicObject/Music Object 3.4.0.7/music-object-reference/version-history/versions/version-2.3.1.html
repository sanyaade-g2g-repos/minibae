Music Object Version 2.3.1
RELEASE DATE: 1998-05-04

What's New In This Version

 IMPORTANT : Since the Beatnik Player 1.3.0 has been released, you can expect that some end-users may be visiting your site with this version. There are issues with the 1.3.0 player and 4.0x versions of Netscape Navigator. You can only avoid these issues by using version 2.3.1 (or higher) of the Music Object API. If you do not upgrade your content to use at least this version, your content may not work reliably for users with Navigator 4.0x and the 1.3.0 Beatnik Player.
BUG FIX - fadeVolume
The fadeVolume method gives authors greater control over volume fades than ever before. In addition to being able to specify a start volume and end volume, the fadeVolume method also allows you to specify a time over which the fade is to occur. One can also register callbacks that are to occur when fade events occur. The FadeEndCallback parameter lets you specify a JavaScript function or statement block that is to be executed when the fade reaches its end. And the FadeStepCallback parameter lets you specify a function or statement block to be executed each time the fade advances. When a fade advances for the last time (when the fade reaches its end), both the fade advance callback and fade end callbacks are executed. Problem is, with earlier versions of Music Object, the final fade advance callback was executed after the fade end callback. This behavior is counter-intuitive and has been corrected. In versions 1.3.1 (and higher) the fade end callback will always get called last.
BUG FIX - engageAudio
In pre-release versions of the Beatnik Player 1.3.0, it was possible to turn the player's underlying audio engine off and on using the disengageAudio and reengageAudio methods. However, right before the time of release, it was decided to roll these two methods into a single engageAudio method that takes a boolean as a parameter. The boolean value indicates whether the engine is to be engaged (true) or disengaged (false). Problem is, the Music Object API was not updated to reflect this last minute change in the player. MusicObject version 2.3.1 fixes this problem.
