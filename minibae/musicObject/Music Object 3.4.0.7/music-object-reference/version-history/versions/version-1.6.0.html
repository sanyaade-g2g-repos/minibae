Music Object Version 1.6.0
RELEASE DATE: 1997-06-26

What's New In This Version

NEW INSTANCE METHOD: playGroovoid
The new playGroovoid instance method provides a convenient way to play Groovoids, simply by specifying the Groovoid name using a string parameter.
NEW INSTANCE METHOD: getInfo
The new getInfo instance method provides a way to obtain specific meta data information from an RMF file that is currently loaded into a Music Object instance.
NEW INSTANCE METHOD: getPanelMode
The new getPanelMode instance method returns a string value, indicating the Music Object instance's current control panel mode.
NEW INSTANCE METHOD: getPanelDisplay
The new getPanelDisplay instance method returns a string value, indicating the Music Object instance's current control panel display type.
API CHANGE: getReverb Becomes getReverbType
It was decided that the name of the method used to query the audio engine's currently selected reverb type should be more specific that just "getReverb". Consequently, the getReverb instance method has been replaced by the functionally equivalent getReverbType instance method.
API CHANGE: setReverb Becomes setReverbType
It was decided that the name of the method used to set the audio engine's global reverb type should be more specific that just "setReverb". Consequently, the setReverb instance method has been replaced by the functionally equivalent setReverbType instance method.
NEW GLOBAL FUNCTION: random
The new random global function provides a convenient way to generate random numbers within a specified range.
NEW GLOBAL FUNCTION: magicEmbed
The new magicEmbed global function provides a convenient way to generate the HTML code necessary to embed a player instance.
NEW GLOBAL FUNCTION: hasBeatnik
The new hasBeatnik global function provides a way to easily determine whether or not the player is installed in the user's browser.
NEW OBJECT: randomPP
The new randomPP object provides a convenient way to generate a sequence of random numbers within a specified range, and where there is a very low probability of any one random number being repeated twice in a row.
GENERAL IMPROVEMENT: Improved Note Playing Methods
The behavior of the noteOn, noteOff, and playNote instance methods has been improved to more correctly implement the channel monophonic state (as set by the setMonophonic instance method and queried by the getMonophonic instance method).
OBSOLETED FEATURE: xFrame Function
The xFrame global function is no longer offered as part of the core Music Object API.
