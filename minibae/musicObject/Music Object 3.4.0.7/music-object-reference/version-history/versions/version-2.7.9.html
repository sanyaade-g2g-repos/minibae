Music Object Version 2.7.9
RELEASE DATE: 1999-01-12

What's New In This Version

NEW CLIENT SUPPORTED - WebTV Satellite Receiver
Version 2.7.9 now supports one of the new models of WebTV set-top boxes, known as the "WebTV Satellite Receiver". This is in addition to the "WebTV Plus Receiver", supported since version 2.7.4. Support for the new WebTV product has not added substantially to the size of the "music-object.js" file.
ISSUE WORKAROUND - Unexplained Long Pause in Internet Explorer

An issue was discovered with Microsoft Internet Explorer and the Beatnik ActiveX control. Occasionally, certain pages would pause for a very long time and not complete loading until something happened in the operating system to "tickle" the browser. The problem was traced to the OBJECT tag's HIDDEN parameter and how this was being used in the dummy instance that is necessary to obtain the version number of the installed player. How exactly this causes the problem is unknown.

The solution, for now, requires that the 1 x 1 pixel dummy player instance not be hidden, resulting in a small "speck", usually towards the top left of any sonified document. This speck does not appear in Netscape Navigator because this browser provides a clear way to obtain version information of installed plug-ins without the need to create dummy instances.

ISSUE WORKAROUND - Unreliable Triggering of the onReady Event
It was discovered that certain content behaved erratically and would sometimes work, and sometimes would not. Investigation revealed that event triggering from ActiveX controls is sometimes unreliable and there can be cases where the scripting layer is not successfully initialized and ready to receive events, when the control believes it is OK to trigger the events. The workaround that was put in place involves a kind of "handshake", where Music Object informs the control that it has received the onReady event. The control will continue periodically retriggering the event unless it receives this verification from Music Object.
IMPROVEMENT - Full Meta Event Support in Internet Explorer
For a while, it has been unclear how to receive parameters passed by ActiveX controls firing events to the browser's scripting layer. So, while previous versions of Music Object where receiving meta events from the player, the all important meta event type and meta event value parameters were not being received. Version 2.7.9 now contains code to receive these parameters and pass them to the registered meta event handler.
API CHANGE - getPlatform Method Replaced by mo_platform Attribute
For the longest time, the getPlatform method has simply returned the userAgent property of the browser being used. Since this can easily be done using the "navigator.userAgent" property, it was decided to remove this method from the API. A more useful way to determine the class of browser being used is to use Music Object's mo_platform global variable.
OPTIMIZATION - General Improvements
Some optimizations were made to the inner workings of Music Object to increase its efficiency. None of these code optimizations require any changes to authored content.
