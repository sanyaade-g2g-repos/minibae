Music Object Extension - Callbacks (version 1.2.0)
RELEASE DATE: 2000-12-28

What's New In This Version

UPDATE - Music Object 3.3.0 Compatibility 
Changes have been made in this version of this extension to allow it to work correctly with version 3.3.0 of the core Music Object API. To avoid compatibility problems, this version of this extension should not be used with Music Object versions earlier than 3.3.0.
IMPROVEMENT - Smaller File Size
This version of this extension has been modified as necessary in order to reduce the size of its JavaScript file using Beatnik's proprietary code "scrunching" process (see the section Smaller Size JavaScript Files in the version history entry for Music Object 3.3.0).
BUG FIX - Meta Event Support
A bug was found where, unless a meta event callback handler was already registered using the onMetaEvent method, the Callback extension's addOnMetaEvent and addMetaEventHandler methods would have no effect. There is a performance optimization within Music Object that results in the player's meta event reporting mechanism being disabled when no meta event handler is registered. This optimization did not take into account the fact that meta event handlers can also be registered exclusively through the addOnMetaEvent and addMetaEventHandler methods of the Callbacks extension, bypassing the core Music Object onMetaEvent method. This bug has been fixed.
BUG FIX - No Meta Event Parameters When Using addMetaEventHandler
A bug was found where, even when a meta event callback handler was already registered using the onMetaEvent method (see previous bug) and meta event handlers added using the Callback extension's addMetaEventHandler method were being called, no meta event type and value parameters were being passed to the handlers registered using this method. This bug has been fixed.
NEW INSTANCE METHOD - addOnAudibleStateChange
The new addOnAudibleStateChange instance method lets you add an event handler to a Music Object instance's list of handlers for the onAudibleStateChange instance callback type that was introduced in Music Object version 3.3.0.
NEW INSTANCE METHOD - addOnOutputVolumeChange
The new addOnOutputVolumeChange instance method lets you add an event handler to a Music Object instance's list of handlers for the onOutputVolumeChange instance callback type that was introduced in Music Object version 3.3.0.
NEW INSTANCE METHOD - clearOnAudibleStateChange
The new clearOnAudibleStateChange instance method lets you clear a Music Object instance's list of handlers for the onAudibleStateChange instance callback type that was introduced in Music Object version 3.3.0.
NEW INSTANCE METHOD - clearOnOutputVolumeChange
The new clearOnOutputVolumeChange instance method lets you clear a Music Object instance's list of handlers for the onOutputVolumeChange instance callback type that was introduced in Music Object version 3.3.0.
NEW INSTANCE METHOD - removeOnAudibleStateChange
The new removeOnAudibleStateChange instance method lets you remove a specific callback handler (or handlers) from a Music Object instance's list of handlers for the onAudibleStateChange instance callback type that was introduced in Music Object version 3.3.0.
NEW INSTANCE METHOD - removeOnOutputVolumeChange
The new removeOnOutputVolumeChange instance method lets you remove a specific callback handler (or handlers) from a Music Object instance's list of handlers for the onOutputVolumeChange instance callback type that was introduced in Music Object version 3.3.0.
NEW STATIC METHOD - Music.removeCallbackHandlers
The new Music.removeCallbackHandlers static method lets you remove a specific callback handler (or handlers) from a specific static callback type (or types). In particular, this method is useful for removing handlers registered for the onAudioEngagedChange static callback type that was introduced in Music Object version 3.3.0, for which no dedicated static method has been provided for removing handlers.
