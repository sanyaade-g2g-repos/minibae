About This Extension

The Channels extension provides convenient access to all the channel properties that can be controlled through the Music Object API. Most of these properties can be accessed using the core Music Object API's setController and getController methods, so this extension is provided primarily to ease programming and to make sonification code more easily understood. As an example, using the Channels extension's setChannelReverbLevel method has the same effect as using the setController method and specifying the value 91 for the ControllerNumber0to127 parameter.

What's New in the Latest Version

There is a dedicated version history for this extension, where you will be able to find out about the new features and possible issues with the most recent version. The version history for this extension is independent of the version history for the core Music Object API.

Getting This Extension

To get the "music-object-x-channels.js" JavaScript file that implements this extension, simply right-click on this link (click-and-hold for Mac users). Then, from the menu that pops up, select the 'Save Link As...' or 'Save Target As...' option and save the file as "music-object-x-channels.js" into an appropriate folder of the local copy of your Web site.

In case you have problems saving the file using the above link, the "music-object-x-channels.js" file can also be found inside the documentation's "javascript" folder, along with the "music-object.js" file.

Dependencies

In order to use this extension, the following JavaScript file(s) must first be sourced in...

  • music-object.js (version 3.3.0 or higher)

For more information on using extensions, consult the section About Music Object Extensions.

 getChannelInstrumentMode

 
 CATEGORY   Instance Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 File Types Supported   RMF, MIDI 
 

The getChannelInstrumentMode method returns an integer value, indicating the current instrument mode for the specified MIDI channel.

    getChannelInstrumentMode ( MidiChannel1to16 )

    ChannelInstrumentMode0to3

The instrument mode selected for a channel can also be obtained using the getChannelNrpn instance method and specifying the value 640 for the Nrpn0to16383 parameter.

Due to a limitation in current versions of the Beatnik Player, the getChannelInstrumentMode method will only return the last instrument mode that was set using the setChannelInstrumentMode or setChannelNrpn instance methods. Because it is possible for instrument mode to be set by MIDI events that occur within an RMF or MIDI file, or by using the setController instance method of the core Music Object API, it is possible that the getChannelInstrumentMode method could return a value that does not reflect the true instrument mode selected for a channel.

So, if you are going to use this method in an interactive audio application, you need to be sure that the RMF or MIDI content you are using does not contain MIDI events that will set instrument mode on channels where your scripting will also be modifying and querying the instrument mode (using the setChannelInstrumentMode and getChannelInstrumentMode methods), and that you also do not use the setController method for setting a channel's instrument mode.

Take a look at the following example...

/*** sets instrument mode 2 for channel 1, using the setChannelInstrumentMode method ***/
myMusicObject.setChannelInstrumentMode (1,2);

/*** sets instrument mode 3 for channel 1, using the setController method ***/
myMusicObject.setController (1,99,5); // non-registered parameter MSB
myMusicObject.setController (1,98,0); // non-registered parameter LSB
myMusicObject.setController (1,6,3);  // non-registered parameter value

/*** alerts the instrument mode for channel 1 ***/
alert (myMusicObject.getChannelInstrumentMode (1));

After the code in this example has been executed, channel 1 will be set to instrument mode 3. However, because the getChannelInstrumentMode method only returns the last instrument mode set by the setChannelInstrumentMode or setChannelNrpn methods, the value 2 will be displayed by the "alert" statement, and not the value 3 as one might expect. This is a limitation that you should be aware of.


  • For more comprehensive information on the Beatnik Player's Channel Instrument Mode feature, consult the section Important Concepts: Channel Instrument Mode.
  • See also the getChannelNrpn and setChannelInstrumentMode instance methods.

 getChannelNrpn

 
 CATEGORY   Instance Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 File Types Supported   RMF, MIDI 
 

The getChannelNrpn method returns an integer value, indicating the last value that was set for the specified non-registered parameter number (NRPN) on the specified MIDI channel.

    getChannelNrpn ( MidiChannel1to16 , Nrpn0to16383 )

    NrpnValue0to127

Values can be set for non-registered parameter numbers via scripting using the setChannelNrpn instance method.

Due to a limitation in current versions of the Beatnik Player, the getChannelNrpn method will only return the last value that was set for an NRPN using the setChannelNrpn instance method (or the setChannelInstrumentMode instance method, in the case of NRPN 640). Because it is possible for NRPN's to be set by MIDI events that occur within an RMF or MIDI file, or by using the setController instance method of the core Music Object API, it is possible that the getChannelNrpn method could return a value that does not reflect the true value of the specified NRPN for a channel.

So, if you are going to use this method in an interactive audio application, you need to be sure that the RMF or MIDI content you are using does not contain MIDI events that will set the values of the NRPN's you will be using in your application, on channels where your scripting will also be modifying and querying these NRPN values (using the setChannelNrpn and getChannelNrpn methods), and that you also do not use the setController method for setting an NRPN's value.

Take a look at the following example...

/*** sets channel 1's instrument mode (NRPN 640) to 2, using the setChannelNrpn method ***/
myMusicObject.setChannelNrpn (1,640,2);

/*** sets channel 1's instrument mode (NRPN 640) to 3, using the setController method ***/
myMusicObject.setController (1,99,5); // non-registered parameter MSB
myMusicObject.setController (1,98,0); // non-registered parameter LSB
myMusicObject.setController (1,6,3);  // non-registered parameter value

/*** alerts channel 1's instrument mode (NRPN 640) setting ***/
alert (myMusicObject.getChannelNrpn (1,640));

After the code in this example has been executed, channel 1 will be set to instrument mode 3. However, because the getChannelNrpn method only returns the last NRPN value set by the setChannelNrpn method (or the setChannelInstrumentMode method, in the case of NRPN 640), the value 2 will be displayed by the "alert" statement, and not the value 3 as one might expect. This is a limitation that you should be aware of.


  • See also the setChannelNrpn, getChannelInstrumentMode, setChannelInstrumentMode, and setController instance methods.

The getChannelPitchBendRange method returns an integer value, indicating the current pitch bend range (measured in musical intervals) set for the specified MIDI channel.

    getChannelPitchBendRange ( MidiChannel1to16 )

    ChannelPitchBendRangeINT

The pitch bend range selected for a channel can also be obtained using the getChannelRpn instance method and specifying the value 0 for the Rpn0to16383 parameter.

Due to a limitation in current versions of the Beatnik Player, the getChannelPitchBendRange method will only return the last pitch bend range that was set using the setChannelPitchBendRange or setChannelRpn instance methods. Because it is possible for pitch bend range to be set by MIDI events that occur within an RMF or MIDI file, or by using the setController instance method of the core Music Object API, it is possible that the getChannelPitchBendRange method could return a value that does not reflect the true pitch bend range selected for a channel.

So, if you are going to use this method in an interactive audio application, you need to be sure that the RMF or MIDI content you are using does not contain MIDI events that will set pitch bend range on channels where your scripting will also be modifying and querying the pitch bend range (using the setChannelPitchBendRange and getChannelPitchBendRange methods), and that you also do not use the setController method for setting a channel's pitch bend range.

Take a look at the following example...

/*** sets channel 1's pitch bend range (RPN 0) to 12, using the setChannelRpn method ***/
myMusicObject.setChannelRpn (1,0,12);

/*** sets channel 1's pitch bend range (RPN 0) to 7, using the setController method ***/
myMusicObject.setController (1,101,0); // registered parameter MSB
myMusicObject.setController (1,100,0); // registered parameter LSB
myMusicObject.setController (1,6,7);   // registered parameter value

/*** alerts channel 1's pitch bend range (RPN 0) setting ***/
alert (myMusicObject.getChannelRpn (1,0));

After the code in this example has been executed, the pitch bend range for channel 1 will be set to 7 musical intervals (a musical 5th). However, because the getChannelRpn method only returns the last RPN value set by the setChannelPitchBendRange or setChannelRpn methods, the value 12 (a full octave) will be displayed by the "alert" statement, and not the value 7 as one might expect. This is a limitation that you should be aware of.


  • See also the getChannelRpn and setChannelPitchBendRange instance methods.

 resetAllControllers

 
 CATEGORY   Instance Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 File Types Supported   RMF, MIDI 
 

The resetAllControllers method lets you easily reset all the controller values for the specified MIDI channel back to their default settings.

    resetAllControllers ( MidiChannel0to16 )

Using the resetAllControllers method has the same effect as using the setController method and specifying the value 121 for the ControllerNumber0to127 parameter, and the value 0 for the ControllerValue0to127 parameter.

For example, the code...

myMusicObject.resetAllControllers (3);

...is equivalent to the code...

myMusicObject.setController (3,121,0);

Because the resetAllControllers method accepts a MidiChannel0to16 parameter, where the value 0 represents all MIDI channels (see the section Important Concepts: Magical MIDI Channel Number 0 and Track Number 0), it is possible to reset all continuous controllers for all MIDI channels of a Music Object instance with a single statement of the form myMusicObject.resetAllControllers (0);.

 setChannelInstrumentMode

 
 CATEGORY   Instance Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 File Types Supported   RMF, MIDI 
 

The setChannelInstrumentMode instance method lets you select the instrument mode for the specified MIDI channel.

    setChannelInstrumentMode ( MidiChannel0to16 , ChannelInstrumentMode0to3 )

It is also possible to set instrument mode for a channel using the setChannelNrpn instance method, or the setController instance method of the core Music Object API.

The following sample code examples all have the effect of selecting instrument mode 2 for MIDI channel 3...

Using the setController Method
myMusicObject.setController (3,99,5); // non-registered parameter MSB
myMusicObject.setController (3,98,0); // non-registered parameter LSB
myMusicObject.setController (3,6,2);  // non-registered parameter value
Using the setChannelNrpn Method
myMusicObject.setChannelNrpn (3,640,2);
Using the setChannelInstrumentMode Method
myMusicObject.setChannelInstrumentMode (3,2);

In each of the sample code listings above, the highlighted code indicates where the instrument mode value is specified.

Due to a limitation in current versions of the Beatnik Player, the getChannelInstrumentMode instance method will only return the last instrument mode that was set using the setChannelInstrumentMode or setChannelNrpn instance methods. So, if you wish to be able to query the instrument mode for a channel using the getChannelInstrumentMode method, you should use either the setChannelInstrumentMode or setChannelNrpn methods for setting instrument mode, and you should avoid using the setController method for this purpose.

In each of the sample code listings above, the highlighted code indicates where the pitch bend range value is specified.

Due to a limitation in current versions of the Beatnik Player, the getChannelPitchBendRange instance method will only return the last pitch bend range that was set using the setChannelPitchBendRange or setChannelRpn instance methods. So, if you wish to be able to query the pitch bend range for a channel using the getChannelPitchBendRange method, you should use either the setChannelPitchBendRange or setChannelRpn methods for setting pitch bend range, and you should avoid using the setController method for this purpose.

The setChannelRpn method lets you set a value for the specified registered parameter number (RPN) on the specified MIDI channel.

    setChannelRpn ( MidiChannel0to16 , Rpn0to16383 , RpnValue0to127 )

Values for registered parameter numbers can be queried via scripting using the getChannelRpn instance method. It is also possible to set values for RPN's using the setController instance method of the core Music Object API.

The following sample code examples both have the effect of setting 12 for the pitch bend range (RPN 0) of MIDI channel 3...

Using the setController Method
myMusicObject.setController (3,101,0); // registered parameter MSB
myMusicObject.setController (3,100,0); // registered parameter LSB
myMusicObject.setController (3,6,12);  // registered parameter value
Using the setChannelRpn Method
myMusicObject.setChannelRpn (3,0,12);

In each of the sample code listings above, the highlighted code indicates where the instrument mode value is specified.

Due to a limitation in current versions of the Beatnik Player, the getChannelRpn instance method will only return the last value that was set for the specified RPN using the setChannelRpn instance method (or the setChannelPitchBendRange instance method, in the case of RPN 0). So, if you wish to be able to query the value of an RPN for a channel using the getChannelRpn method, you should use either the setChannelRpn method (or the setChannelPitchBendRange method, in the case of RPN 0) for setting the value of the RPN, and you should avoid using the setController method for this purpose.

Static Methods

 Music.getControllerName

 
 CATEGORY   Static Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 

The Music.getControllerName static method returns a string value, representing the name of the specified MIDI continuous controller number.

    getControllerName ( ControllerNumber0to127 )

    ControllerNameSTR

The following table shows all the possible values that can be returned by this method...

Method Call Return Value
Music.getControllerName (0) "Bank Number"
Music.getControllerName (1) "Modulation"
Music.getControllerName (7) "Volume"
Music.getControllerName (10) "Pan"
Music.getControllerName (11) "Expression"
Music.getControllerName (91) "Reverb Level"
Music.getControllerName (93) "Chorus Level"

As a rule, symbolic controller names are only returned for MIDI controller numbers for which the Channels extension provides get- and set- style methods (such as getChannelExpression and setChannelExpression).

There may be other MIDI controllers that are supported by the player (see the appendix Supported MIDI Controllers) for which the Music.getControllerName method will not return a dedicated symbolic name. One such example is MIDI controller number 98, which is the "Non-registered Parameter Number LSB" controller. There are also a good number of MIDI controllers that are not supported by the player. If, for whatever reason, the Music.getControllerName method does not know the symbolic name of the specified MIDI controller number, then the method will simply return the specified MIDI controller number as a string in the form "MIDI Controller [ControllerNumber0to127]". So, the statement Music.getControllerName (98); would return the value "MIDI Controller 98".

 Music.midiMsbLsbToInt

 
 CATEGORY   Static Method 
 MODULE   Music Object Extension - Channels 
 Min. Player Version   1.0.0 
 

The Music.midiMsbLsbToInt static method lets you convert a MIDI MSB-LSB pair to an integer value.

    Music.midiMsbLsbToInt ( Msb0to127 , Lsb0to127 )

    Integer0to16383

With the MIDI communication protocol, many values are constrained to a single 7-bit byte (where one bit is reserved as a flag). This means, of course, that these values are constrained to the range of 0 through 127. It is no coincidence that values for MIDI continuous controllers, such as channel volume, are constrained to this range. And it is no coincidence that note numbers for MIDI note events are constrained to this range. And it is also no coincidence that there are 128 MIDI continuous controllers in total.

Now, in order to offer a large set of registered parameter numbers (RPN) and non-registered parameter numbers (NRPN), so that these parameters don't suffer the same ultimate scarcity as the rather limited set of 128 MIDI continuous controllers, it was decided to use two 7-bit bytes for specifying RPN's and NRPN's. These two bytes are combined together to form a 14-bit number. One byte, called the MSB (Most Significant Byte), is used to represent the upper 7 bits, while the other byte, called the LSB (Least Significant Byte), is used to represent the lower 7 bits.

For example, an MSB-LSB pair where the MSB value is 5 and where the LSB value is 0 would represent the number 640. NRPN 640 is the NRPN that is used to set a channel's instrument mode (see the section Important Concepts: Channel Instrument Mode). If you have used RPN's or NRPN's in music compositions, then you will most probably have had plenty of experience specifying these parameters by their MSB-LSB pair in your music sequencer's MIDI event view. Because all the methods in this extension that deal with registered and non-registered parameter numbers take a single integer value for specifying these parameters' numbers, the Music.midiMsbLsbToInt method is provided as a convenience in case you have become accustomed to thinking of RPN's and NRPN's in terms of their MSB-LSB pair.

To illustrate our example for NRPN 640, the code...

myMusicObject.setChannelNrpn (1,Music.midiMsbLsbToInt (5,0),2);

...is equivalent to the code...

myMusicObject.setChannelNrpn (1,640,2);
