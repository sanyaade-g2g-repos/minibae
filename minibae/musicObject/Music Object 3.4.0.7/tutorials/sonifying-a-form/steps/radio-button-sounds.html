STEP 4: Adding the Radio Button Sounds

Now, to add the radio button sounds. Unlike checkboxes, radio buttons cannot be checked and unchecked by repeatedly clicking on them. So, in our sonification we will make clicking on a radio button produce only one possible sound. We should choose a Groovoid that sounds "sharp" and "delicate" - something that is in keeping with the small point-like nature of a radio button. The Groovoid should still sound reasonably decisive, like the checkbox check sound we used in the previous step. Enter, UI-Click2.

Try fooling around with the radio buttons in the form below. They are now sonified, in addition to the checkboxes which are already sonified from the previous step. As you interact with the form, alternate between clicking on radio buttons and checkboxes to get a feel for the different sonification effects and how they contrast with each other yet also compliment each other.

Just like the previous step, where we sonified checkboxes by adding the checkboxSound function, we have now added the new radioButtonSound function for sonifying our form's radio buttons. However, unlike the checkboxSound function, the radioButtonSound function is extremely simple. All it does is play a Groovoid through the "formSounds" Music Object instance. In fact, we could just as easily have placed the code that is inside the radioButtonSound function inside each of the radio button's ONCLICK event handlers and then dispensed with the additional function entirely. So, why define a custom function?

Well, it is common wisdom in the field of programming to move repeated code segments into separate function definitions and then simply call that function in all the places where you would have repeated the same code segment. This makes it really easy to make one simple change in a single function and have that change be shared by all the places where that function is called. To use our own example, if we wanted to easily experiment with Groovoids to choose just the right sound for the radio buttons, we could very easily try different Groovoids in the radioButtonSound function, each time reloading the page after we had modified the function.

Using this separate function approach also allows us to build a more complex sonification behavior for radio buttons if we care to, much like the checkboxSound function. In this event, sharing a sonification behavior that's implemented in a custom function not only becomes a big timesaver, but also represents a greater efficiency in code size.
