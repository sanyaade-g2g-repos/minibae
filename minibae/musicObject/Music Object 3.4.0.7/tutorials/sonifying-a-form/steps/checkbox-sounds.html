It has that feeling of something being undone or erased - wiped clean.

Most significant in the source code listing for this step is the addition of the checkboxSound function. This function is responsible for actually playing the sound when a user clicks on a checkbox, and takes care of deciding whether a "check" or "uncheck" sound should be played, based upon the current state of the checkbox that is being clicked. The function can achieve this dual-purpose behavior because the one parameter that it is receiving, checkboxID, is a handle to the exact checkbox form element being clicked. The function can use this handle to find the new state of the checkbox and play the appropriate Groovoid for either a checked state or an unchecked state.

Now, in order to "wire" each checkbox in the form to play sounds using this function, an ONCLICK event handler has been added to every checkbox in the form. These event handlers are easy to spot in the source code listing, since they have been highlighted in red. Notice how each call to the checkboxSound function is passing this as a parameter. The reserved identifier this in JavaScript is a handle to the current object, which in the scope of a form element's event handler will be a handle to the form element for which the event is occurring. This gives us an easy way to pass the checkboxSound function a handle to the checkbox that is being clicked.

The checkboxSound function then uses the checked property of the checkbox form element to determine if the checkbox has been checked or unchecked. An appropriate Groovoid is then played through the Music Object instance "formSounds" using the play method, and passing just a Groovoid name as a parameter.
