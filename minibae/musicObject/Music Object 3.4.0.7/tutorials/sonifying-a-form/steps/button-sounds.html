STEP 7: Adding the Submit and Reset Button Sounds

We will play the Groovoid "Fanfare-Mars" when the user chooses to clear their entries in the form by using the reset button. This Groovoid has the feel of making an error or starting over. By contrast, we will play the Groovoid "Fanfare-CorpSting1" when the user proceeds with their purchase by clicking the form's submit button. This Groovoid has a pleasant, conclusive, and almost congratulatory feel to it.

In order to play the sounds for the reset and submit buttons, we have added the formResetSound and formSubmitSound functions. In addition, we have added ONCLICK event handlers to the reset and submit buttons to call these functions.

Now, the formResetSound is very simple and just plays the Groovoid "Fanfare-Mars" through the "formSounds" Music Object instance. On the other hand, the formSubmitSound function has a little added intelligence to it. Because the process of submitting a form results in the current page being flushed so that the results of a cgi script, for instance, can be loaded into the browser, we need to create a mechanism to ensure that the submission of the form is deferred until the Groovoid finishes playing.

To achieve this behavior, we must first add a "return false" statement to the end of the ONCLICK event handler for the submit button. This tells the browser that it should not continue to execute the submission action once the ONCLICK event handler has completed. Now, because we are effectively cancelling the form submission in the submit button's ONCLICK event handler, we must make sure that the formSubmitSound function somehow triggers the form's submission action.

Because we want to have the entire "Fanfare-CorpSting1" Groovoid play through to the end before the form submission takes place, we will use the onStop event handler for the "formSounds" Music Object instance to trigger the submission action. This is done in the statement formSounds.onPlay ("formSounds.onStop ('document." + formElement.form.name + ".submit ()')");.
