Beatnik Player's Embedding Properties

A Beatnik Player is embedded on a page using either an EMBED tag, an OBJECT tag, an EMBED/OBJECT tag combination, or Music Object's magicEmbed method. There are many properties that can be used to customize the appearance and behavior of a Beatnik Player instance. This reference describes all the properties that can be set when embedding Beatnik Player instances on a page.

  • Some properties described in this reference are specific to the Beatnik plug-in (Netscape Navigator), and others are specific to the Beatnik ActiveX Control (Microsoft Internet Explorer). This will be noted in the reference information for each property.

  • Some of the properties are required, while others are optional. For instance, it is required that every EMBED and OBJECT tag on a page have the WIDTH and HEIGHT properties specified. If a particular property is required it will be noted in the reference for that property.

  • For further guidance on using the properties described in this reference to create your own EMBED / OBJECT tags, consult the tutorial Putting Music on a Page.

The ALIGN property lets you specify how a player should be aligned against other HTML elements in a page's layout.

    ALIGN={LEFT | RIGHT | CENTER | TOP | BOTTOM | MIDDLE}

The ALIGN property can be used as an attribute for the EMBED tag, an attribute for the OBJECT tag, and also a property for the magicEmbed method. Because the browser handles alignment of the player instance, the flow of HTML around the player instance will follow the same rules of alignment that are used for any other HTML elements, such as IMG tags, for example.

 AUTOSTART

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes 
 OBJECT Tag Attribute   - 
 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   1.0.0 
 Default Value   TRUE 
 

The AUTOSTART property lets you specify whether or not the player should start playing automatically once it completes loading in the file specified by the SRC property.

    AUTOSTART={TRUE | FALSE}

The AUTOSTART property can be used as an attribute for the EMBED tag, a parameter for the OBJECT tag, and also a property for the magicEmbed method. When this property is set to true, the file will play immediately upon loading, and when this property is set to false, the file will not automatically begin playing. All files that are subsequently played by the player instance using the Music Object play method will observe the value set for this property. The autostart state can be obtained and modified using the Music Object API's getAutostart and setAutostart methods, respectively.


  • default is TRUE if the AUTOSTART property is specified without a value, and also TRUE if the property is not present at all
  • attribute of the EMBED tag and parameter of the OBJECT tag
  • can be used in Music Object's magicEmbed method

The CLASSID property lets you coerce Microsoft Internet Explorer into using the Beatnik Player to embed an audio file.

    CLASSID="CLSID:B384F118-18EE-11D1-95C8-00A024330339"

This property is specific to Microsoft Internet Explorer and is used to specify the unique Windows class ID of the Beatnik ActiveX control. This property can be used to force the Beatnik Player to handle a file that is specified in the OBJECT tag's SRC parameter, in much the same way as the TYPE attribute is used with the EMBED tag. In order to invoke the Beatnik ActiveX control, the value specified for the CLASSID property must be exactly as shown above.

It is not necessary to specify this property if you are using the Music Object API's magicEmbed method (or other embedding methods) to embed player instances, as these methods set this property automatically.

The DISPLAY property lets you specify the display type for the player instance's control panel.

    DISPLAY={SONG | SYSTEM}

Using this property, you can specify whether the player instance's control panel should display information specific to the currently loaded file, or information that is global in nature (such as the output meters or oscilloscope). The DISPLAY property can be used as an attribute for the EMBED tag, a parameter for the OBJECT tag, and also a property for the magicEmbed method. For an understanding of how different values for this property relate to different control panel layouts, consult the appendix Control Panel Layouts.


  • control panel display type can also be set via scripting using the setPanelDisplay method, and can be queried using the getPanelDisplay method
  • refer further to the appendix Control Panel Layouts

 FILEVOLUMESCALING

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes 
 OBJECT Tag Attribute   - 
 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   2.1.0 
 Default Value   TRUE 
 

The FILEVOLUMESCALING property lets you specify whether a file's internal volume setting should be scaled against the player volume, or simply be ignored.

    FILEVOLUMESCALING={TRUE | FALSE | ON | OFF | 1 | 0}

The values TRUE, ON, and 1 all have the same meaning and indicate that file volume scaling should be enabled. Similarly, the values FALSE, OFF, and 0 all have the same meaning and indicate that file volume scaling should be disabled. For a better understanding of file volume scaling, consult the section Important Concepts: File Volume Scaling.

 GROOVOID

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes

 Min. Player Version   1.0.0

The GROOVOID property lets you specify the name of a Groovoid that should be played in place of the file specified by the SRC property.

    GROOVOID="GroovoidNameSTR"

A Groovoid is a "pre-cooked" musical piece that is built right into the player's soundbank. For more information on Groovoids, consult the section Important Concepts: Groovoids. When using the GROOVOID property, the SRC property should still be used to specify the location of a stub file. This is to maintain backward compatibility with some older player versions that always required a SRC property to be specified - even when a Groovoid was being embedded. Newer versions of the player will ignore the SRC property if a GROOVOID property is specified, anyway.


  • a stub file must still be specified in the SRC property
  • the GROOVOID property should be used in conjunction with the TYPE and SRC properties

The HEIGHT property lets you specify the height of the player instance's control panel, measured in pixels.

  • HEIGHT must still be specified even if the HIDDEN property is present
 HIDDEN

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes 
 OBJECT Tag Attribute   - 
 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   1.0.0 
 Default Value   FALSE 
 

The HIDDEN property lets you specify that the player instance's control panel should be hidden.

    HIDDEN

Embedding hidden player instances is especially useful for sonifying Web sites where you don't want the sonification to take up any of the sites' precious graphic real estate. When using the HIDDEN attribute, one should also specify the WIDTH=0 HEIGHT=0 embedding properties so that the hidden player instance doesn't occupy any significant space within the page layout. For instance, if you specified the embedding properties WIDTH=144 HEIGHT=60 HIDDEN, the embedded player instance would indeed be hidden, but would still create a blank 144 x 60 pixel space in the page layout - not really the desired effect.


  • WIDTH=0 HEIGHT=0 should also be specified when using the HIDDEN property

 LOOP

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes 
 OBJECT Tag Attribute   - 
 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   1.0.0

The LOOP property lets you specify the desired looping behavior for the player instance.

    LOOP={TRUE | ON | FALSE | OFF | integer where 0 is infinite}

When a value of true, on, or 0, is specified for this property, then the player will loop the file it is playing indefinitely. When a value of false, off, or 1 is specified, then the player will play through the file it is playing only once and will not repeat. When an integer greater than 1 is specified for this property, then the player will play through the file it is playing as many times as is specified by the property.

The LOOP property can be used as an attribute for the EMBED tag, a parameter for the OBJECT tag, and also a property for the magicEmbed method. All files that are subsequently played by the player instance using the Music Object play method will observe the value set for this property. The looping state can be obtained and modified using the Music Object API's getLoop and setLoop methods, respectively.


  • default is TRUE if the LOOP attribute is specified without a value, and FALSE if the attribute is not present at all
 MODE

 
 CATEGORY   Embedding Property 
 EMBED Tag Attribute   yes 
 OBJECT Tag Attribute   - 
 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   1.0.0 
 Default Value   SCOPE 
 

The MODE property lets you specify the DISPLAY mode of the information portion of the player instance's control panel.

    MODE={METERS | SCOPE | COPYRIGHT}

When a value of SONG is specified for the DISPLAY property, then there is only one display mode available - the COPYRIGHT mode - which shows the song's author, copyright, and publisher information. When a value of SYSTEM is specified for the DISPLAY property, then there are two display modes available - METERS and SCOPE.

Depending on the values for the WIDTH and HEIGHT properties, there may or may not be an information portion present in the player instance's control panel layout. If this portion is not present, then setting this property will have no effect. The MODE property can be used as an attribute for the EMBED tag, a parameter for the OBJECT tag, and also a property for the magicEmbed method.

For an understanding of how different values for this property relate to different control panel layouts, consult the appendix Control Panel Layouts.


  • control panel display mode can also be set via scripting using the setPanelMode method, and can be queried via scripting using the getPanelMode method
  • refer further to the appendix Control Panel Layouts

The PLUGINSPAGE property lets you specify the URL location of the Beatnik Player download page.

    PLUGINSPAGE="http://www.beatnik.com/to/?player"

By using this property, you can direct Web site visitors who don't already have the Beatnik Player installed straight to the Beatnik Player download page. This is especially important if you are authoring a Web site to specifically make use of the features of the Beatnik Player. This property is specific to Netscape Navigator and Netscape Communicator. In order for the installation of the Beatnik Player to work correctly, the value specified for the PLUGINSPAGE property must be exactly as shown above.

It is not necessary to specify this property if you are using the Music Object API's magicEmbed method (or other embedding methods) to embed player instances, as these methods set this property automatically.

 SRC

 
 CATEGORY   Embedding Property

 OBJECT Tag PARAM   yes 
 magicEmbed Property   yes 
 Min. Player Version   1.0.0

The SRC property lets you specify the URL location of a music or audio file that should be loaded into the player instance.

    SRC=FileUrlSTR

The file specified by this property can be a resource on the Web or on a local hard drive or other storage device. The file can be any one of the Beatnik Player's supported file types. New files can be loaded into the player instance subsequently via scripting, using the Music Object API's play method. This property should also be used when the GROOVOID property is specified, so as to maintain backward compatibility with some older player versions.

The TYPE property lets you coerce Netscape Navigator and Netscape Communicator into using the Beatnik Player to embed an audio file.

    TYPE="audio/rmf"

This property is specific to Netscape Navigator and Netscape Communicator and can be used to force the Beatnik Plug-in to handle a file that is specified in the EMBED tag's SRC attribute, in much the same way as the CLASSID attribute is used with the OBJECT tag. Using this property, one may eliminate the need to setup special MIME types on one's server.

The VOLUME property lets you specify the playback volume level for the player instance.

    VOLUME=Volume0to100

Player volume level is measured in percent, where a value of 100 represents maximum volume and a value of 0 represents minimum volume (or silence). For more information, see the Volume0to100 parameter.

The VOLUME property can be used as an attribute for the EMBED tag, a parameter for the OBJECT tag, and also a property for the magicEmbed method. This property can also be set via scripting using the Music Object API's setVolume method, or queried via scripting using the getVolume method.
