STEP 9: Using the sonifyFormsWithHandlers Method

Up until now in this tutorial we've shown how a form can be sonified by explicitly adding event handlers to all of the elements of the form. Now, in a complex form, not only can this add quite a bit of HTML and JavaScript code to your page, but it also creates a fair amount of work for you. What if there were a wondrous and magical way to sonify a form without even touching the form's HTML? Too good to be true, right? Well, get ready to act pleasantly surprised.

At this point we'll depart into some wicked DOM (Document Object Model) Voodoo and make our lives one whole helluva lot easier. "Document Object Model?" you ask. Well, if you've ever coded an image rollover behavior for a page in a Web site, then you've already been exposed to the DOM. Basically, browsers expose all the HTML elements on a page to the JavaScript scripting environment. This is referred to as "reflection", since the HTML elements on a page are "reflected" to JavaScript in the form of objects.

Wizard to the Rescue

Now, because elements of an HTML page can be accessed using JavaScript, it becomes possible to write JavaScript code to automatically scan for specific types of elements on an HTML page and, if desired, actually modify the properties of any elements. This principle can be applied quite effectively to form elements, where it is possible using JavaScript to scan a document for all form elements and add event handlers as needed to sonify the elements.

We're not going to get into the frightening details of how this is done, because Beatnik, Inc. has already gone to the trouble of creating just the code needed to automatically sonify all elements of all forms on any HTML page. This code is neatly packaged into the Music Object Extension - Wizards module. What we will be looking at is how to use the methods provided in this extension to conveniently sonify forms in various ways. We will be continuing with the same form as we've used in the earlier steps of this tutorial, and we will show how the same results from the earlier steps can be achieved with less work using "Music Object Extension - Wizards".

In this first step using the extension, we will use Groovoids to achieve the same level of sonification as we saw in the earlier step Adding the Submit and Reset Button Sounds. And we will achieve these same results with much less sonification code, thanks to the Music.sonifyFormsWithHandlers static method found in this extension.

The Music.sonifyFormsWithHandlers method lets you sonify all the elements of all forms on any HTML page using handlers. All you need to do is supply the method with the desired handlers that are to be executed for each type of form element event that can occur. For this method, handlers must be string parameters containing the JavaScript code that is to be executed when a specific event occurs. For instance, in this page's example we have used the handler 'formSounds.play ("UI-Click7")' for when checkboxes get clicked.

There are many form element events that can be sonified automatically using the Music.sonifyFormsWithHandlers method, and a parameter is provided for each of these events, as follows...

  • CheckboxCheckJavaScriptSTR
    ...for when a checkbox, as defined by the "<INPUT TYPE=checkbox>" tag, becomes checked.

  • CheckboxUncheckJavaScriptSTR
    ...for when a checkbox becomes unchecked.

  • RadioButtonClickJavaScriptSTR
    ...for when a radio button, as defined by the "<INPUT TYPE=radio>" tag, is clicked.

  • ListFocusJavaScriptSTR
    ...for when a drop-down listbox, as defined by the SELECT and OPTION tags, becomes focused.

  • ListChooseJavaScriptSTR
    ...for when a new item is selected from a drop-down listbox.

  • ListBlurJavaScriptSTR
    ...for when a drop-down listbox loses its focus.

  • MultilistFocusJavaScriptSTR
    ...for when a multiple selection listbox, as defined by the SELECT and OPTION tags, becomes focused.

  • MultilistChooseJavaScriptSTR
    ...for when an item is selected or unselected in a multiple selection listbox.

  • MultilistBlurJavaScriptSTR
    ...for when a multiple selection listbox loses its focus.

  • TextboxFocusJavaScriptSTR
    ...for when a textbox, as defined by the "<INPUT TYPE=text>", or a textarea, as defined by the TEXTAREA tag, becomes focused.

  • TextboxBlurJavaScriptSTR
    ...for when a textbox or textarea loses its focus.

  • ButtonClickJavaScriptSTR
    ...for when a button, as defined by the "<INPUT TYPE=button>" tag, is clicked.

  • FormResetJavaScriptSTR
    ...for when a form reset button, as defined by the "<INPUT TYPE=reset>" tag, is clicked.

  • FormSubmitJavaScriptSTR
    ...for when a form submit button, as defined by the "<INPUT TYPE=submit>" tag, is clicked.

In the source code listing above, all the code needed for sonification of the form is highlighted in red. One of the first things you should notice, towards the top of the source code, is how the JavaScript file "music-object-x-wizards.js" that implements Music Object Extension - Wizards has been sourced into the page immediately following the sourcing in of the "music-object.js" file. It is important that Music Object extensions always be sourced after the core "music-object.js" file has first been sourced in, otherwise the extensions will not work.

Towards the bottom of the source code listing you will see the block of code (highlighted in red) that is responsible for actually sonifying the form. Three key things are taking place in this block of code...

  1. A Music Object instance is created so that it can be used to play Groovoids for the sonification of the form elements. The instance is created in the statement "formSounds = new Music ();".

  2. The sonifyForms function is defined, which gets called when the document has finished loading in (see the BODY tag's ONLOAD event handler towards the top of the example). Inside the implementation for this function there is only one statement - a call to the Music.sonifyFormsWithHandlers method, which is responsible for automatically adding the appropriate event handlers to all elements of the form.
     IMPORTANT : It's very important that the methods inside the "Music Object Extension - Wizards" module only be called once a document has finished loading in, as calling these methods while a document is still loading may, in some cases, cause unpredictable results.

    Now, remember the long list of parameters we discussed earlier? This is an example of those parameters in action. Wherever there is a form element event to be sonified you will see a string parameter containing a JavaScript statement that plays a Groovoid through the Music Object instance "formSounds". And wherever a form element event is not to be sonified you will see an empty string.

    In case you're finding it a little hard to map the parameter values in this method call to the parameters we discussed earlier, consider the following table...
