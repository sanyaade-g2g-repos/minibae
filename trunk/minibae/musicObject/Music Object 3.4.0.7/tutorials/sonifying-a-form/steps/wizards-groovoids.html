STEP 10: Using the sonifyFormsWithGroovoids Method

Now, to make your life even easier, the Music.sonifyFormsWithGroovoids static method lets us achieve exactly the same results as in the earlier step Using the sonifyFormsWithHandlers Method, but with even simpler code!

Just like the Music.sonifyFormsWithHandlers method, the Music.sonifyFormsWithGroovoids method lets you sonify all the elements of all forms on any HTML page. All you need to do is supply the method with the desired Groovoids that are to play for each type of form element event that can occur. For instance, in this page's example we have used the Groovoid "UI-Click7" for when checkboxes get clicked.

Try out the form below. You should notice that it sounds exactly the same as the earlier step that used the Music.sonifyFormsWithHandlers method.

As with the Music.sonifyFormsWithHandlers method, a parameter is provided for each of the form element events that can be sonified automatically using Groovoids. The parameters are as follows...

  • CheckboxCheckGroovoidNameSTR
    ...for when a checkbox, as defined by the "<INPUT TYPE=checkbox>" tag, becomes checked.

  • CheckboxUncheckGroovoidNameSTR
    ...for when a checkbox becomes unchecked.

  • RadioButtonClickGroovoidNameSTR
    ...for when a radio button, as defined by the "<INPUT TYPE=radio>" tag, is clicked.

  • ListFocusGroovoidNameSTR
    ...for when a drop-down listbox, as defined by the SELECT and OPTION tags, becomes focused.

  • ListChooseGroovoidNameSTR
    ...for when a new item is selected from a drop-down listbox.

  • ListBlurGroovoidNameSTR
    ...for when a drop-down listbox loses its focus.

  • MultilistFocusGroovoidNameSTR
    ...for when a multiple selection listbox, as defined by the SELECT and OPTION tags, becomes focused.

  • MultilistChooseGroovoidNameSTR
    ...for when an item is selected or unselected in a multiple selection listbox.

  • MultilistBlurGroovoidNameSTR
    ...for when a multiple selection listbox loses its focus.

  • TextboxFocusGroovoidNameSTR
    ...for when a textbox, as defined by the "<INPUT TYPE=text>", or a textarea, as defined by the TEXTAREA tag, becomes focused.

  • TextboxBlurGroovoidNameSTR
    ...for when a textbox or textarea loses its focus.

  • ButtonClickGroovoidNameSTR
    ...for when a button, as defined by the "<INPUT TYPE=button>" tag, is clicked.

  • FormResetGroovoidNameSTR
    ...for when a form reset button, as defined by the "<INPUT TYPE=reset>" tag, is clicked.

  • FormSubmitGroovoidNameSTR
    ...for when a form submit button, as defined by the "<INPUT TYPE=submit>" tag, is clicked.

In the source code listing above, all the code needed for sonification of the form is highlighted in red. You may wish to refer back to the source code listing for the earlier step, Using the sonifyFormsWithHandlers Method, for a comparison of the Music.sonifyFormsWithGroovoids and Music.sonifyFormsWithHandlers methods. One of the first things you will notice is how the Music.sonifyFormsWithGroovoids takes care of managing the Music Object instance's play state, so that it is not necessary as it was using the Music.sonifyFormsWithHandlers to provide the code to stop the currently playing Groovoid for form elements events that were to have no sonification. With the Music.sonifyFormsWithGroovoids, all you have to do is specify an empty string for events that are to be silent.

Towards the bottom of the source code listing you will see the block of code (highlighted in red) that is responsible for actually sonifying the form. The basic approach used in this example is the same as that used in the earlier step, Using the sonifyFormsWithHandlers Method, so we won't discuss things in detail again. In summary, the three key things that are taking place in this block of code are...

  1. A Music Object instance is created, to be used to play Groovoids through.
  2. The sonifyForms function is defined, which will be called when the document finishes loading in.
  3. A Beatnik Player instance is embedded for the Music Object instance "formSounds".
