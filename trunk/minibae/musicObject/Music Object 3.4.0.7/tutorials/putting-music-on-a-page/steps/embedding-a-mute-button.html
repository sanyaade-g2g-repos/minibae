STEP 6: Embedding a Mute Button

It is not always necessary for a Beatnik Player to actually be playing music. When the control panel is set in SYSTEM display, then the mute button that appears in the top right corner will affect the Beatnik Player's global audio output. This includes all other player instances, regardless of whether or not they are even in the same document or window as the instance with the mute button.

In this example, we have used this principle to our advantage by embedding one player instance that serves purely as a common global mute button and does nothing more. Try playing some of the files embedded below by using the transport controls. Then, click on the mute button. You will notice that the music you started playing now becomes silent, and the mute button changes state to indicate that the Beatnik Player's global audio output has been muted. Now, if you click on the mute button a second time the global mute will once again be turned off, the mute button graphic will return to its original unmuted state, and the music will continue playing. Each time you click on the mute button the global mute state will toggle.

Truly Global

To drive home the point that the mute button affects the global audio output of the Beatnik Player, try playing several of the files simultaneously before clicking on the mute button. Sure, you'll have to endure a moment's cacophony, but it will be perfectly clear when you click the mute button that it is affecting the combined audio output of all Beatnik Player instances.

Now, for each of the player control panels, pop up the player's contextual menu and select the "System Display" option. The contextual menu can be popped open on Windows systems by right-clicking on the control panel, and on MacOS systems by clicking and holding on the control panel. Once all of the control panels are set to the system display, each control panel should have a mute button. When you click on a mute button on any one of the control panels, you will notice how the mute buttons for all other control panels are updated to the new global mute state.

In order to make the control panel only as big as the mute button, we first set the HEIGHT to 15 so that the control panel only displays the slim top portion. We then take advantage of a subtle little behavior of the control panel. That is, when the WIDTH is set to less than 144, the mute button of the control panel is always drawn against the right-hand side. So, while a WIDTH of 40 does not give the control panel enough room to display both the Beatnik symbol and the mute button that usually appear in the top portion of the SYSTEM display, it still gives the control panel enough room to display at least the mute button.

In order to illustrate this principle better, take a look at the control panels below where the WIDTH is progressively decreased, starting from 144 and ending at 41. At a WIDTH of 41, you can still see a tiny smidgen of the Beatnik "B" peeping through with player versions earlier than 2.1. But, with Beatnik Player versions 2.1 and higher, the mute button is clean.

Differences Between Player Versions

With Beatnik Player versions earlier than 2.1, the mute button is shown as a speaker icon that becomes grayed out when the Beatnik Player's global audio output is muted. With versions 2.1 and higher, the mute button is shown as a pair of green and red on-off lights, where the red light is lit when the audio is muted and the green light is lit when the audio is not muted. With both styles of mute buttons, clicking anywhere on the mute button has the effects of toggling the global audio output's mute state.
