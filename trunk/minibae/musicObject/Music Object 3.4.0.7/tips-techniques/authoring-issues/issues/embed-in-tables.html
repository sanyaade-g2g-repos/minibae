Using EMBED in tables

Synopsis

Sometimes plug-ins that are embedded in table layouts will play perfectly but will not respond to Music Object scripting.

Symptoms

As in the Using EMBED in multiple frames issue, there seems to be a failure in LiveConnect to successfully reflect plug-in instances to the JavaScript level. In cases where the plug-in reflection fails, it is not possible to communicate to a plug-in instance through its symbolic handle (as defined in the NAME attribute). The plug-in will still play files, however.

Cause

According to Netscape engineers, "Two-pass tables duplicate their objects in JS, in 3.0, for images, applets, and embeds. These related bad bugs should be fixed in 4.0, finally. The standard workaround is to use document.embedName or document.embeds['name'] rather than document.embeds[i]."

The unfortunate news is that we had no success with the document.embedName approach, either.

Solution

Without a full understanding of the mechanics behind this failure in LiveConnect, and given the fact that the problem can occur even in single level tables in a frameset (not nested tables), our advice is to completely avoid placing plug-in instances in table layouts. This is the safest way to avoid this issue.
