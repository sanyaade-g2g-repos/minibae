DEMO: The noteOn, noteOff, and playNote Methods

The Music Object API's noteOn, noteOff, and playNote instance methods let you trigger and stop musical notes from virtually any user interaction within a page. And the great thing about musical notes is that they are free, with no downloads of audio or music files required. The instrument sounds, like the Groovoids, are built directly into the Beatnik Player's soundbank.

So Many Sounds, So Few Buttons!

Musical notes play through the Beatnik Player's software synthesizer, on any one of the many instrument sounds in the player's soundbank. In case you thought that 72 Groovoids was a lot, the Beatnik Player soundbank contains an incredible 256 musical instrument sounds, as well as 109 percussion sounds. The percussion sounds are particularly useful for sonifying user interface elements on Web pages - ideal for link click sounds, button click sounds, link mouseover sounds, form submit sounds, and more.

Check out the buttons below. Each button has a different click sound, thanks to sounds chosen from the Beatnik Player's percussion banks. To the right of each button is the source code of the playNote method call for each button. Go on and click 'em...

The Highs and the Lows

Your palette of sounds gets extended even further when you take musical instrument sounds and play them at different pitches. Each instrument sound can be played at 128 different pitches!

The following buttons all play the Beatnik Player's castanets percussion sound. However, completely different effects are achieved simply by playing the musical notes at different pitches. Using only one sound from the soundbank, each button now has its own distinctive click effect. To the right of each button is the code used to produce the sound for the button.

Roll Your Own Behaviors

With the Music Object API, you get full control to make the kind of musical behaviors that you want for your Web page elements. Combine the noteOn, noteOff, and playNote methods to create sonified links in an enormous number of different ways.

Take a look at the links below, for intance. Each link has its own unique sound for when you "mouse over" and for when you click. The "mouse over" sound plays for as long as you are over the link, and then stops as soon as you move the mouse away from the link. The "mouse over" sound also stops when you click on the link and the click sound is played. To the right of each sonified link is the source code used to create the musical behavior for the link. Try it out...

It All Stacks Up

Think that all the possibilities have been exhausted? Think again. There's no saying that you only need to play one musical note at a time. Even more complex textures can be achieved when you combine several musical notes, playing at different pitches, and playing different instrument sounds. You can even make your sonified elements musically correct by playing proper musical chords. Heck, you could even make it so that your Web page elements play chords that are in the same key as a piece of background music you are playing on the same page!!

Of course, there's nothing to say that each one of the musical notes that makes up a chord should play exactly the same instrument sound. You can, most certainly, achieve a stunning variety of different sound textures by combining just the right instrument sounds at just the right pitches.

The Chords Extension

While you can use multiple calls to the noteOn or playNote methods in order to trigger the combinations of notes that make up certain chords, the Music Object API makes it even easier to play chords using the dedicated Chords extension. The playChord instance method, provided in this extension, lets you play musical chords by simply specifying chord names - you don't even have to specify the individual notes that make up the chords.

For a demonstration of the Chords extension in action, see the Chords extension demo.

Randomize It!!!

Because the Music Object API is designed for a scripting environment, you can build up more complex behaviors using its basic methods. For example, you could quite easily create your own function for playing random musical notes within certain specified parameters. Beatnik Inc. has even gone to the trouble of offering more advanced features like this in ready-made Music Object extensions, such as Music Object Extension - Dynamic which offers a playRandomNote method.

To get an idea of the other cool things that can be done using musical notes, see the playRandomNote demo.
