Auto Install

Web content authors are understandably concerned about any reliance their content may have upon browser add-on components shipped by third party developers. For one thing, such add-on components have traditionally required the user to download large installers and then interrupt their browsing in order to install the component into the browser. Not only would this installation potentially take a long time, but the user may not even return immediately to the Web site they were visiting after installation is complete.

Well, with the advent of the 4.x browsers (Netscape Navigator 4.x and Microsoft Internet Explorer 4.x), it is now possible for users who do not have the Beatnik Player installed to install it while still using their browser. It is no longer necessary for users to download a large manual installer, quit their browser, locate and run the downloaded installer, then relaunch their browser and attempt to continue browsing where they left off.

Beatnik Inc. offers an easy online way to install the Beatnik Player - the Beatnik Player Install Wizard - which works for Netscape Navigator and Microsoft Internet Explorer. By using Music Object, sonified pages can invoke the Beatnik Player Install Wizard automatically without even taking the user away from the Web page they're on at the time. A small prompt window (shown below) opens up to take the user step-by-step through the installation process. At any time, the user has the option of cancelling the install and may still browse the Web page they were on - just without any Beatnik music or audio functionality.

Don't Bug the User

Another concern that some Web content developers have is that they do not wish to interrupt the Web browsing experience for users - even if a user needs to have a certain component installed in order to get a richer experience. To such content developers, site sonification is seen as an enhancement that is of secondary importance to the basic function that the Web site provides. These Web developers may be quite happy knowing that the investment of time and energy put into sonifying the Web content will provide a richer experience for those users who have the Beatnik Player installed, but may be lost on those users who do not have the Player installed.

In cases like this where not interfering in any way with the user's browsing of the site is of paramount importance, it is possible to suppress the default behavior of Music Object by setting the Music.showCompatibilityPrompt property to a value of false subsequent to sourcing in the "music-object.js" file, as follows...

<SCRIPT SRC="music-object.js"></SCRIPT>

<SCRIPT LANGUAGE=JavaScript><!-- //

   Music.showCompatibilityPrompt = false;

// --></SCRIPT>

<!-- The rest of your document follows.... -->

When a sonified Web page loads and Music Object detects that the Beatnik Player is not installed on a user's system and the Music.showCompatibilityPrompt property is set to false, the auto install prompt will not be displayed. Instead, the page will still behave normally except there will not be any music or audio functionality through Beatnik.

  Setting this property to false will also suppress an auto install prompt that may result from the sonified content requiring a version of the Beatnik Player that is greater than the version currently installed in the user's browser. See the Cutting Edge Content section for more information.
Filling the Holes

So, Music Object has the ability to detect if the Beatnik Player is installed or not. But, what happens when you're embedding the Beatnik Player onto a page and you've specified the control panel to be visible? If the Beatnik Player is not installed, what does the user see instead? Well, since maintaining the Web page's layout integrity is of key importance, it wouldn't really be acceptable for nothing to be put in place of the Player control panel - only to have the rest of the document's layout offset as a result. Nor would it be acceptable for there simply to be an inexplicable blank hole left in the page, with the user having no clue as to the purpose of the hole.

So, Music Object fills the space that would otherwise be occupied by the Player control panel with a table of the exact same dimensions, as specified by the WIDTH and HEIGHT attributes. In cases where the user does not have the Beatnik Player installed at all, the table will contain the text "Get Beatnik!", as shown in the figure to the right. The text will be linked so that clicking on it will invoke the Beatnik Player Install Wizard.

In cases where the user has a version of the Beatnik Player installed, but that version is lower than a minimum version required by the sonified content and the Music.silentIfInadequate static property is set to true, then the text "Upgrade Beatnik!" will be displayed as shown in the figure to the right. The text will be linked to invoke the Beatnik Player Install Wizard.


  • the color for the substitute table is always white.
  • the table is sized to take up the same amount of space in the layout as the Player control panel would have taken.
  • the table will not be shown if the HIDDEN attribute is specified when embedding the instance.
  • the table will also not be shown if the user's browser / platform combination is not supported by the Beatnik Player, or if it is supported but the Beatnik Player Install Wizard is not able to update the user's browser - such as is the case with WebTV.
  • if the specified WIDTH is less than 90 or the specified HEIGHT is less than 15, then the table and linked text will not be displayed, but a blank table with no text will be created to take up the same amount of space as specified by the WIDTH and HEIGHT attributes.
  • if the specified HEIGHT is less than 30, then the linked text inside the table will be scaled down to a smaller font size in order to still fit.
Control Panel Substitute Graphics

In addition to the built-in mechanism that creates a substitute "Get Beatnik!" table in cases where the Beatnik Player is not installed on the user's system, or a substitute "Upgrade Beatnik" table when the version installed is not adequate for the sonified content, Music Object also provides the option of displaying substitute graphics in place of Player control panels. Unlike the substitute table mechanism which is self-contained inside Music Object, using the substitute graphics mechanism requires you to serve additional graphics from your Web site. You can serve these graphics from anywhere on your Web site, as long as you specify the correct path to these graphics from the current HTML page.

An Example

Take a look at the example below. In order to use the substitute graphics mechanism, we simply define the static property Music.getPlayerImagesPath subsequent to sourcing in the "music-object.js" file, as follows...

<SCRIPT SRC="music-object.js"></SCRIPT>

<SCRIPT LANGUAGE=JavaScript><!-- //

   Music.getPlayerImagesPath = "http://www.my-web-site.com/my-images-folder/";

// --></SCRIPT>

<!-- The rest of your document follows.... -->

The Music.getPlayerImagesPath static property is a String variable specifying the Internet location of the graphics that are to be used as substitutes for the control panel. You do not name the individual graphics - just the directory in which they reside. Music Object expects that the graphics have very specific names, as listed in the table further below. If you do not name your graphics exactly as required, then the substitute mechanism will not work and the user will see broken graphics symbols on the page where the Player control panel was to be. Note that the directory path you specify must also end with a forward slash ("/").

In our example, we have specified an absolute URL. Of course, you could also specify a relative URL, but the advantage of specifying an absolute URL in this case is that you can copy the exact same line of code into all your sonified HTML pages. In fact, you could even put this statement into a general Music Object preferences file that is sourced into all sonified documents, in much the same way as the shared "debug-prefs.js" file discussed in the Debugging Aids section.

Substitute Graphics Filenames

The filenames required for the various substitute graphics are shown in the table below, along with their exact dimensions. When you serve these graphic files from your own Web site, it is imperative that you name them exactly the same as shown here.

Image Filename Width Height
get-player-144x60.gif 144 60
get-player-144x45.gif 144 45
get-player-144x15.gif 144 15


  • If you name the images incorrectly in your Web site media, the mechanism will not work.
  • If you specify an incorrect path to the images from the sonified document, the mechanism will not work.
  • If you're using WIDTH and HEIGHT attribute values other than those listed in the table, the substitute graphics mechanism will not be used, but the substitute table mechanism will be used instead.
  • Unlike the substitute table mechanism, there is no provision for a variation on the graphics for when the user has an installed version of the Beatnik Player that is lower than a version required by the content - the same graphic will be used for both the "Get Beatnik!" and "Upgrade Beatnik" situations.
Your Own Look and Feel

You are free to craft your own Beatnik Player substitute graphics. After all, it's your Web site and the graphics that appear in your pages should match the look and feel of the other graphics in your pages, as well as the site's overall aesthetic. If you create your own substitute graphics, you need to make sure that they are in the GIF format and are named exactly as shown in the table above, matching even the case of the letters.

Cutting Edge Content

At some point you may find yourself in a position where you're creating content that uses features of the most recent version of the Beatnik Player. Your content may be so dependent upon certain new features that you may not wish the user to experience the content unless they have the very latest version installed. In this event, the Music.requireMinPlayerVersion static method lets you specify a minimum version that is needed in order for the content to be viewed. If the user doesn't have at least this version installed when they encounter your sonified content, a prompt window (seen below) will pop up prompting them to upgrade their version of the Player.

Also, if the Music.silentIfInadequate static property is set to true, then no Beatnik Player instances will be embedded in the page and the page will have no music or audio functionality through Beatnik. Instead, either substitute tables or a substitute graphics will be shown in the place of all Player control panels that would otherwise have appeared on the page. See the Filling the Holes and Control Panel Substitute Graphics sections for more information.

An Example

The Music.requireMinPlayerVersion static method lets you specify the minimum version of the Player that is required in order for the content to function correctly. Typically, you will specify the version number as three digits separated by dots (periods), as in "1.3.2". The first digit represents the major version number, the second digit represents the minor version number, and the third digit represents a bug fix or maintenance release number. Using the Music.requireMinPlayerVersion static method is easy. Simply call the method subsequent to sourcing in the "music-object.js" file, as follows...

<SCRIPT SRC="music-object.js"></SCRIPT>

<SCRIPT LANGUAGE=JavaScript><!-- //

   Music.requireMinPlayerVersion ('2.0.0');

// --></SCRIPT>

<!-- The rest of your document follows.... -->


  • when trying to ensure feature support, it is best not to specify the third digit (bug fix / maintenance release number) in your required minimum version, as this number may vary significantly between different SKU's of the Beatnik Player - such as the Beatnik Player for Netscape Navigator (Windows), the Beatnik Player for Microsoft Internet Explorer (Windows), the Beatnik Player PRO for Netscape Navigator (MacOS), etc.
  • the "upgrade" prompt will not be shown if the Music.showCompatibilityPrompt static property is set to false. See the Don't Bug the User section for more information.
