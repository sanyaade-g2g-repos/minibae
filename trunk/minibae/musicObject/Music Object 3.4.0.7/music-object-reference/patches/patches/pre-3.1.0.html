About This Patch

If you've been authoring using a version of Music Object before 3.1.0 and would now like to upgrade to the latest version, there may be some features of the older version of Music Object that you were using that are no longer supported in the recent version. Several changes to the API were made between version 3.0.0 and version 3.1.0. You should consult the Music Object Version History to find out more about the changes in version 3.1.0 and how they may affect your code.

If you find that your code will, indeed, be affected by the changes in the API, then you have two options...

  1. make the necessary changes to your code and switch over to version 3.1.0, or...
  2. switch over to version 3.1.0 without changing your code and use this extension to mask the differences so that your code still works.

Of course, if none of the changes between versions 3.0.x and 3.1.0 affect your code, then you can make the switch to 3.1.0 without worries.

Among the things that this patch does,...

  • It defines the musicObject constructor which was available in Music Object versions before 3.1.0. Starting from version 3.1.0, the constructor was changed to Music.
  • It defines the redundant instance methods fadeFromTo, getVersion, isReady, and playGroovoid which were available in Music Object versions before 3.1.0 and are now no longer available in versions 3.1.0 and higher.
  • It defines the instance methods getNoteName, getNoteNumber, and stopAll, which were available in Music Object versions before 3.1.0 and are now only available as static methods in versions 3.1.0 and higher.
  • It defines the mo_installBeatnik global function which was available in Music Object versions before 3.1.0 and has been replaced by the Music.installPlayer static method in versions 3.1.0 and higher.
Getting This Patch

To get the "music-object-p-pre310.js" JavaScript file that implements this patch, simply right-click on this link (click-and-hold for Mac users). Then, from the menu that pops up, select the 'Save Link As...' or 'Save Target As...' option and save the file as "music-object-p-pre310.js" into an appropriate folder of the local copy of your Web site.

In case you have problems saving the file using the above link, the "music-object-p-pre310.js" file can also be found inside the documentation's "javascript" folder, along with the "music-object.js" file.

Dependencies

In order to use this patch, the following JavaScript files must first be sourced in...

  • music-object.js (version 3.1.0 or higher)
