About This Extension

Unlike the getChannelMute, getChannelSolo, getTrackMute, getTrackSolo, setChannelMute, setChannelSolo, setTrackMute, and setTrackSolo methods provided by the core Music Object API that only allow one to test and affect the state of one channel or track at a time, the additional methods in this extension allow one to test and affect the state of multiple channels or tracks in a single operation.

Consider the following before-and-after example. Using the core Music Object API's setTrackMute method, it takes a chunk of code to mute 8 tracks. However, using the setTracksMute method provided in this extension, the same code can be neatly folded into a single statement - nice and clean.

BEFORE (Using setTrackMute)
myMusicObject.setTrackMute (1,true);
myMusicObject.setTrackMute (7,true);
myMusicObject.setTrackMute (15,true);
myMusicObject.setTrackMute (24,true);
myMusicObject.setTrackMute (42,true);
myMusicObject.setTrackMute (43,true);
myMusicObject.setTrackMute (47,true);
myMusicObject.setTrackMute (59,true);
AFTER (Using setTracksMute)
myMusicObject.setTracksMute ('1,7,15,24,42,43,47,59',true);

The methods provided by this extension make it easier to program Web-based "remix" interfaces, where musical compositions are "performed" as the user switches on and switches off different parts of the music - these parts often being made up of multiple channels or tracks.

What's New in the Latest Version

There is a dedicated version history for this extension, where you will be able to find out about the new features and possible issues with the most recent version. The version history for this extension is independent of the version history for the core Music Object API.

Getting This Extension

To get the "music-object-x-groups.js" JavaScript file that implements this extension, simply right-click on this link (click-and-hold for Mac users). Then, from the menu that pops up, select the 'Save Link As...' or 'Save Target As...' option and save the file as "music-object-x-groups.js" into an appropriate folder of the local copy of your Web site.

In case you have problems saving the file using the above link, the "music-object-x-groups.js" file can also be found inside the documentation's "javascript" folder, along with the "music-object.js" file.

Dependencies

In order to use this extension, the following JavaScript file(s) must first be sourced in...

  • music-object.js (version 3.3.0 or higher)

For more information on using extensions, consult the section About Music Object Extensions.

The getChannelsMute method determines the overall channel mute state for multiple channels in one simple step.

    getChannelsMute ( MidiChannelGroupSTR )

    MuteStateBOOL

The getChannelsMute method returns a boolean value representing the overall mute state of the specified channel group. If all channels in the specified group are muted, then the group is regarded as being muted and the getChannelsMute method returns a value of true. However, if any or all of the channels in the group are not muted, then the group is not regarded as being muted and a value of false is returned. It is possible to toggle the mute state of a channel group with a statement such as...

myMusicObject.setChannelsMute ('1,4,7,15',!myMusicObject.getChannelsMute ('1,4,7,15'));

This is not the same as the statement myMusicObject.setChannelsMute ('1,4,7,15','Toggle'), since the Toggle value will have the effect of toggling the mute states of each of the individual channels that make up the group, thereby "inverting" the mute state of the group. It is, however, equivalent to the statement myMusicObject.setChannelsMute ('1,4,7,15','Toggle Group'), since using the Toggle Group value results in the setChannelsMute method first determining the mute state of the group as a whole, and then setting all the specified channels to the opposite state.


  • getChannelsMute ( MidiChannel0to16 )

    When the MidiChannel0to16 parameter is used in place of the MidiChannelGroupSTR parameter, then the getChannelsMute method can be used in just the same way as the getChannelMute method, making the getChannelsMute method more versatile, because you could pass it a parameter that is a Number representing a single MIDI channel, or a String representing a group of MIDI channels.


  • Compare to the getChannelMute method.

The setChannelsMute method lets you set the channel mute state for multiple channels in one simple step.

    setChannelsMute ( MidiChannelGroupSTR , MuteStateBOOL )

The setChannelsMute method lets you set the mute state of multiple MIDI channels in one simple step. This lets you treat multiple MIDI channels together as a group. This is quite useful, among other things, for coding Web-based "remix" interfaces. A MIDI channel group is specified as a string value, containing a comma-separated list of MIDI channel numbers, as in myMusicObject.setChannelsMute ('1,3,4,8,10,11',true).

By using one of its variations, the setChannelsMute method also lets you use a convenient Toggle value to automatically invert the mute states of all channels in a group, as in myMusicObject.setChannelsMute ('1,3,4,8,10,11','Toggle').

One can also use the Toggle Group value to automatically toggle the mute state of the group as a whole, as in myMusicObject.setChannelsMute ('1,3,4,8,10,11','Toggle Group'). After using the Toggle Group value, all channels in the group will have the same mute state - either muted or unmuted - depending on the original overall mute state of the group.


  • setChannelsMute ( MidiChannelGroupSTR , MuteStateBOOL , ExclusiveBOOL )

    When a value of true is specified for the optional ExclusiveBOOL parameter, then all channels other than those specified in the MidiChannelGroupSTR parameter will be set to the opposite state of the channels in the group. So, the statement myMusicObject.setChannelsMute ('1,3,4,8,10,11',true,true) will mute the channels 1, 3, 4, 8, 10, and 11, and unmute the channels 2, 5, 6, 7, 9, 12, 13, 14, 15, and 16. The ExclusiveBOOL parameter may be used optionally with all other variations of this method.

  • setChannelsMute ( MidiChannelGroupSTR , MuteStateSTR )

    When the MuteStateSTR parameter is used in place of the MuteStateBOOL parameter, it becomes possible to make use of the convenient Toggle value, which makes it easy to toggle the mute states of each channel in a group in one simple step, as in the statement myMusicObject.setChannelsMute ('1,3,7,8,11','Toggle')

  • setChannelsMute ( MidiChannel0to16 , MuteStateBOOLorMuteStateSTR )

    When the MidiChannel0to16 parameter is used in place of the MidiChannelGroupSTR parameter, then the setChannelsMute method can be used to affect the state of a single channel in the same way as the setChannelMute method. So, the setChannelsMute method is more versatile, because you could pass it a parameter that is a Number representing a single MIDI channel, or a String representing a group of MIDI channels. This also makes the convenient Toggle value available when simply affecting the state of one MIDI channel, as in myMusicObject.setChannelsMute (1,'Toggle').

  • setChannelsMute ( MidiChannelGroupSTRorMidiChannel0to16 )

    When no MuteStateBOOL or MuteStateSTR parameter is specified, then the setChannelsMute method will mute all the channels in the specified MIDI channel group. So, the statement myMusicObject.setChannelsMute ('1,3,4,8,10,11') would have the same effect as the statement myMusicObject.setChannelsMute ('1,3,4,8,10,11',true).


  • Compare to the setChannelMute method.
