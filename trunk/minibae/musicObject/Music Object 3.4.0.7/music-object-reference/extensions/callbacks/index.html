About This Extension

Unlike the onLoad, onMetaEvent, onPause, onPlay, onReady, and onStop methods that provide the core Music Object API's callback support, these additional methods allow one to stack multiple event handlers for the same player event.

For example, while the onPlay method only allows one to override the currently registered handler for the play event, the addOnPlay method lets one add new handlers on top of currently registered handlers. Then, when a play event occurs for a Music Object instance, all handlers that have been added to the stack using the addOnPlay method will be executed in the order in which they were added to the stack. In fact, callback handlers that are added for any callback type are executed when the callback event occurs, and in the order in which they were added to the stack of callback handlers. Consider the following code example...

myMusicObject.addOnPlay (initialiseUI);
myMusicObject.addOnPlay ('fileStarted = true');
myMusicObject.addOnPlay ('preloadNextSong ("creepy-swamp.rmf")');

Now, when a play event occurs for myMusicObject, the initialiseUI function will first be called, then the statement fileStarted = true will be evaluated, and then the statement preloadNextSong ("creepy-swamp.rmf") will be evaluated. Of course, in this very straightforward situation one could simply have rolled all the actions into a single myOnPlayHandler function and registered this using the onPlay method, but the goal of the methods in this extension is to allow multiple code modules to add to the callback handler stack for any given callback type, without needing to worry about handlers that have already been registered by other code modules.

One of the things that this allows is for multiple code segments to cooperatively listen for events that occur within a shared Music Object instance, without these code modules having to have an intimate understanding of each other's internal implementation. Using the methods for managing callbacks that are provided in this extension, different code modules can use the callbacks for a Music Object instance without overriding each other's registered callback handlers, as would otherwise occur with the core Music Object callback methods.

Tip on Removing Callbacks

If a function, rather than a statement block, has been registered as a callback handler, then that callback handler can only be removed by specifying the original function that was registered as the callback handler, and not a statement block representing a call to the function. For instance, the following code will not work correctly...

  INCORRECT
function myOnPlayHandler () {
	// code to handle onPlay event
}

myMusicObject.addOnPlay (myOnPlayHandler);
myMusicObject.removeOnPlay ('myOnPlayHandler ()');

However, the code listed below is correct and the handler registered using the addOnPlay method is successfully removed using the removeOnPlay method.

  CORRECT
function myOnPlayHandler () {
	// code to handle onPlay event
}

myMusicObject.addOnPlay (myOnPlayHandler);
myMusicObject.removeOnPlay (myOnPlayHandler);

Getting This Extension

To get the "music-object-x-callbacks.js" JavaScript file that implements this extension, simply right-click on this link (click-and-hold for Mac users). Then, from the menu that pops up, select the 'Save Link As...' or 'Save Target As...' option and save the file as "music-object-x-callbacks.js" into an appropriate folder of the local copy of your Web site.

In case you have problems saving the file using the above link, the "music-object-x-callbacks.js" file can also be found inside the documentation's "javascript" folder, along with the "music-object.js" file.

Dependencies

In order to use this extension, the following JavaScript file(s) must first be sourced in...

The addMetaEventHandler instance method lets one add event handlers for specific meta events that are encountered within an RMF or MIDI file.

    addMetaEventHandler ( MetaEventTypeSTR , MetaEventValueSTR , HandlerFunction )

Unique meta events are identified by their meta event type and meta event value, as specified by the MetaEventTypeSTR and MetaEventValueSTR parameters, respectively. So, rather than registering a generalized handler using the onMetaEvent or addOnMetaEvent methods and then writing your own code for the handler to detect specific meta events that you care about, you can instead let this extension do some of this hard work for you by registering handlers for the specific events that you care about.

For instance, if you wanted to be notified each time an RMF composition reached the beginning of a fade out at the end of the composition, and assuming that the composition contains a marker meta event at the start of this fade out, then one could use the following code to register the handler for this unique meta event...

function musicFadeOutBegins () {
   // code to change UI to reflect fade out
}

myMusicObject.addMetaEventHandler ('Marker','Fade to end',musicFadeOutBegins);

The addOnAudibleStateChange instance method lets you add an event handler to a Music Object instance's list of audible state change event handlers.

    addOnAudibleStateChange ( HandlerFunction )

The addOnAudibleStateChange method is similar to the onAudibleStateChange method in that it lets you register handlers for the audible state change event of a Music Object instance. The key difference is that, while the onAudibleStateChange method only lets you register one handler for the audible state change event and overrides any existing registered handler, the addOnAudibleStateChange method instead lets you add to a stack of event handlers for the audible state change event. When an audible state change event occurs for a Music Object instance, all handlers that have been added to the stack using the addOnAudibleStateChange method will be executed in the order in which they were added to the stack.

For backward compatibility reasons, handlers registered for the audible state change event using this method do not override a handler registered using the core Music Object API's onAudibleStateChange method. Any handler registered for the audible state change event using the onAudibleStateChange method is executed before any handlers that have been added to the handler stack using the addOnAudibleStateChange method.

The clearCallbacks instance method lets you clear all callbacks that have been assigned to a Music Object instance.

    clearCallbacks ( )

The clearCallbacks method clears all callbacks that have been registered for the Music Object instance. This includes callbacks registered using the core Music Object API's onLoad, onMetaEvent, onPause, onPlay, onReady, and onStop methods, as well as this extension's addMetaEventHandler, addOnLoad, addOnMetaEvent, addOnPause, addOnPlay, addOnReady, and addOnStop methods. If you do not wish to affect handlers registered using the core Music Object API's callback methods, then you should use the removeCallbackHandlers method.

    clearOnAudibleStateChange ( )

The clearOnAudibleStateChange method clears all handlers that have been added for the audible state change event using this extension's addOnAudibleStateChange method, as well as the core Music Object API's addCallbackHandler and onAudibleStateChange methods. To remove only those handlers that have been added using the addOnAudibleStateChange or addCallbackHandler methods, use the variation of the removeOnAudibleStateChange method that takes no parameters, as in the statement myMusicObject.removeOnAudibleStateChange ( ). It should be noted that using the statement onAudibleStateChange (null) to clear a handler that has been registered for the audible state change event using the onAudibleStateChange method has no effect on handlers that have been registered using the addOnAudibleStateChange or addCallbackHandler methods.

Removing Event Handlers

 removeCallbackHandlers

 
 CATEGORY   Instance Method 
 MODULE   Music Object Extension - Callbacks 
 Min. Player Version   1.0.0 
 File Types Supported   RMF, MIDI, WAV, AIFF, AU 
 

The removeCallbackHandlers instance method lets you remove the specified callback handler (or handlers) from the specified instance callback type (or types).

    removeCallbackHandlers ( InstanceCallbackTypeSTR , HandlerFunction )

The removeCallbackHandlers method lets you remove a specific callback handler from the stack of handlers that have been registered for the specified instance callback type. This is equivalent to using some of the other methods in this extension, such as removeOnLoad, removeOnMetaEvent, removeOnPause, removeOnPlay, removeOnReady, and removeOnStop. For instance, the statement myMusicObject.removeCallbackHandlers ('onPlay',myOnPlayHandler) is equivalent to the statement myMusicObject.removeOnPlay (myOnPlayHandler).

In order to successfully remove a specific handler using the removeCallbackHandlers method, one must correctly specify the callback type and the callback handler that were used when initially registering the callback handler. For example, in order the remove the handler registered using the statement myMusicObject.addCallbackHandler ('onPlay',myOnPlayHandler), one should use the statement myMusicObject.removeCallbackHandlers ('onPlay',myOnPlayHandler). In this example, using the statement myMusicObject.removeCallbackHandlers ('onPlay','myOnPlayHandler ()') would be incorrect and would not remove the handler, since the original statement that registered the callback handler registered it using the Function type to specify the handler, and not the String type.

It is also possible to remove all handlers that have been added for a specific callback type by simply not specifying the handler to be removed, as in the example statement myMusicObject.removeCallbackHandlers ('onPlay'), which will remove all handlers that were added using the addOnPlay or addCallbackHandler methods. This is equivalent to the statement myMusicObject.removeOnPlay ().

Finally, one can remove all instance callback handlers that have been added using the methods in this extension by simply not specifying the instance callback type or handler, as in the statement myMusicObject.removeCallbackHandlers ().


  • removeCallbackHandlers ( InstanceCallbackTypeSTR , HandlerJavaScriptSTR )
  • removeCallbackHandlers ( InstanceCallbackTypeSTR )
