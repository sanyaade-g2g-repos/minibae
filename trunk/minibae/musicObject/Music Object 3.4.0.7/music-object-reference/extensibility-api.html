Introduction

Starting from version 3.1, the Music Object API has offered additional functionality in the form of optional Music Object extensions. With the release of version 3.3, the API (Application Programming Interface) for writing these extensions became official and public. This section explains the architecture upon which Music Object extensions, language modules, and patches are based and includes a reference to all the methods and properties of the extensibility API.

An Extension Template

The following sample code listing provides a template that can be used as a starting point for creating your own custom Music Object extensions...

Copy-and-Paste
/**********
     TITLE: Music Object Extension - MyExtensionName
   VERSION: 1.0.0
    AUTHOR: Whoever I Am
 COPYRIGHT: 1999 MyCompany, Inc. All Rights Reserved
  REQUIRES: music-object.js (3.2.0 or higher)
**********/

function mo_myInstanceMethod (someParam1,someParam2,someParamN) {
   // some code to implement this instance method
}

function mo_myStaticMethod (someParam1,someParam2,someParamN) {
   // some code to implement this static method
}

/*** Instance Extenders ***/

function mo_addMyExtensionInstanceProperties (ID) {
   ID.myExtensionInstanceProperty1 = false;
   ID.myExtensionInstanceProperty2 = 0;
   //...
   //...
   ID.myExtensionInstancePropertyN = null;
}

Music.addInstanceExtender (mo_addMyExtensionInstanceProperties);

/*** Public Instance Methods ***/

Music.addPublicInstanceMethods ('myInstanceMethod');

/*** Public Static Methods ***/

Music.addPublicStaticMethods ('myStaticMethod');

/*** Public Static Properties ***/

Music.myExtensionStaticProperty1 = false;
Music.myExtensionStaticProperty2 = 0;
//...
//...
Music.myExtensionStaticPropertyN = null;

 Music.addEventHandler

 Music.addKnownClient

 
 CATEGORY   Extensibility API Static Method 
 MODULE   Music Object Core
 Music Object Component - Compatibility 
 

The Music.addKnownClient static method lets you add a client profile to the list of supported clients, with the assumption that the extension calling this method may implement code necessary to support the client.

    Music.addKnownClient ( ClientNameSTR , ClientPlatformSTR , ClientMinVersionSTR , ClientIsUpgradableBOOL )


  • Music.addKnownClient ( ClientNameSTR , ClientPlatformSTR , ClientMinVersionSTR , ClientIsUpgradableBOOL , CustomPropertiesOBJ )

    When the optional CustomPropertiesOBJ parameter is specified, then all the properties of the object specified by the CustomPropertiesOBJ parameter will be copied to the object that is created by the Music.addKnownClient method. Among other things, this feature can be used to add notes regarding issues or limitations with the support for a specific known client, or to add notes regarding why a specific known client is not supported. Custom properties that are stored with a known client are simply ignored by the core Music Object API, and so it is up to the code that registeres such custom properties to make use of them as appropriate.

    When specifying a value for the CustomPropertiesOBJ parameter, it is possible to create the object containing the custom properties directly in the call to the Music.addKnownClient method, as in the statement...

    Music.addKnownClient (
   'Some Browser',
   'Win32,MacPPC',
   '2.0 to 4.1',
   true,
   {
      notes:'Certain maintenance release versions of this browser may not be supported.',
      downloadURL:'http://www.somedomain.com/downloads/player-for-some-browser.cgi'
   }
);
    .

 Music.openWindow

 
 CATEGORY   Extensibility API Static Method 
 MODULE   Music Object Core
 Music Object Component - Compatibility 
 

The Music.openWindow static method lets you open a centered window of the specified size and properties, for the purposes of displaying prompt dialogs or informational messages.

    Music.openWindow (
    FileUrlSTR ,
    WindowNameSTR ,
    WindowWidthINT ,
    WindowHeightINT ,
    ShowStatusBarBOOL ,
    WindowPropertiesSTR ,
    WindowLeftPositionINT ,
    WindowTopPositionINT
)


  • When a value of null is specified for the ShowStatusBarBOOL parameter, then the window that is opened by this method will by default have no status bar.
  • When a value of null is specified for the WindowPropertiesSTR parameter, then an empty string will be used for this parameter.

  • Music.openWindow ( FileUrlSTR , WindowNameSTR , WindowWidthINT , WindowHeightINT , ShowStatusBarBOOL , WindowPropertiesSTR , WindowLeftPositionINT )

    When no WindowTopPositionINT parameter is specified, then the window will be centered vertically on the screen.

  • Music.openWindow ( FileUrlSTR , WindowNameSTR , WindowWidthINT , WindowHeightINT , ShowStatusBarBOOL , WindowPropertiesSTR )

    When there is also no WindowLeftPositionINT parameter specified, then the window will also be centered horizontally on the screen.

  • Music.openWindow ( FileUrlSTR , WindowNameSTR , WindowWidthINT , WindowHeightINT , ShowStatusBarBOOL )

    When there is also no WindowPropertiesSTR parameter specified, then the default window properties for this method will be used.

  • Music.openWindow ( FileUrlSTR , WindowNameSTR , WindowWidthINT , WindowHeightINT )

    When there is also no ShowStatusBarBOOL parameter specified, then the window that is opened by this method will by default have no status bar.


  • The WindowNameSTR parameter may not contain any spaces or non-alphaneumeric characters, excepting the underscore character ("_").

 Music.setClientInfo

 
 CATEGORY   Extensibility API Static Method 
 MODULE   Music Object Core
 Music Object Component - Compatibility 
 

The Music.setClientInfo static method lets you explicitly set the client information that the Music Object compatibility component will use to determine whether or not the client is supported.

    Music.setClientInfo ( AppNameSTR , AppVersionSTR , PlatformSTR , UserAgentSTR )


  • When a value of null is specified for the AppNameSTR parameter, then the value of the browser's navigator.appName property will be used for this parameter.
  • When a value of null is specified for the AppVersionSTR parameter, then the value of the browser's navigator.appVersion property will be used for this parameter.
  • When a value of null is specified for the PlatformSTR parameter, then the value of the browser's navigator.platform property will be used for this parameter, unless the platform can be determined from the value of the UserAgentSTR parameter.
  • When a value of null is specified for the UserAgentSTR parameter, then the value of the browser's navigator.userAgent property will be used for this parameter.

  • Music.setClientInfo ( AppNameSTR , AppVersionSTR , PlatformSTR )

    When no UserAgentSTR parameter is specified, then the value of the browser's navigator.userAgent property will be used for this parameter.

  • Music.setClientInfo ( AppNameSTR , AppVersionSTR )

    When there is also no PlatformSTR parameter specified, then the value of the browser's navigator.platform property will be used for this parameter, unless the platform can be determined from the value of the UserAgentSTR parameter.

  • Music.setClientInfo ( AppNameSTR )

    When there is also no AppVersionSTR parameter specified, then the value of the browser's navigator.appVersion property will be used for this parameter.

  • Music.setClientInfo ( )

    When there is also no AppNameSTR parameter specified, then the value of the browser's navigator.appName property will be used for this parameter.
