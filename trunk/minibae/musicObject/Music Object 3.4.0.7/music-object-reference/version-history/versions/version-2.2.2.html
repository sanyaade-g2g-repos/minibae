Music Object Version 2.2.2
RELEASE DATE: 1998-03-02

What's New In This Version

ISSUE WORKAROUND - Player Volume

It was established that there is a bug in the behavior of the volume attribute that has been frustrating authors. The concept of a player volume level that is maintained across different songs was not behaving as authors would expect. If you set the player volume with the VOLUME attribute and then subsequently set the player's volume by using the setVolume method, any time a new song was started, or a song was stopped and then restarted, the player's volume level would be reset to the initial volume specified in the VOLUME attribute. The expected behavior is that using setVolume would set a new player volume level that would override what the author specified as the initial volume level with the VOLUME attribute.

This version of the Music Object class implements a workaround where a player's volume state is maintained in the Music Object instance you create. The onPlay callback is then used to reset the player's volume to this volume state each time a file begins playing. The volume state that Music Object maintains is initialized with the value a user specifies in the VOLUME attribute (the default is 100). Calls to Music Object methods that affect player volume (such as setVolume, fadeVolume, fadeTo, etc.) will adjust this instance volume state.

This bug has been present ever since the very first version of the Beatnik Player (1.0) and is still present in version 1.3.0. It should be fixed in later versions of the player, but the Music Object workaround will help until this issue is addressed in the player's code.
