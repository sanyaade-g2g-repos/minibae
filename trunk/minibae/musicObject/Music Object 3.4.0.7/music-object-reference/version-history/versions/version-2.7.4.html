Music Object Version 2.7.4
RELEASE DATE: 1998-11-25

What's New In This Version

NEW CLIENT SUPPORTED - WebTV Plus

The significant difference between version 2.7.4 and version 2.6.9 is that this new version achieves compatibility with WebTV Plus set-top boxes. WebTV implemented support for many of Beatnik's music playback capabilities in their WebTV Plus Receiver, including the playback of RMF files. However, certain issues were discovered with the implementation of JavaScript that made it necessary to make significant changes to the Music Object API implementation. These changes are all "under the hood" and don't affect how Beatnik Web content authors use the API - all the methods, functions, properties, etc. are still exactly the same.

One unfortunate result of achieving Music Object support for the WebTV Plus Receiver is that the "music-object.js" file grew in size by roughly 2Kb - still a modest price to pay for content support across more platforms. It is WebTV's intention to eliminate the need for the Music Object workarounds in future versions of the WebTV Plus software. At this stage, it may be possible to remove the workarounds and once again reduce the size of the "music-object.js" file by 2Kb.

IMPROVEMENT - magicEmbed & mo_magicEmbed

It is no longer necessary, with the magicEmbed method and the mo_magicEmbed function, to specify the "<EMBED " text that opens the EMBED tag or the ">" that closes the EMBED tag. This makes use of magicEmbed and mo_magicEmbed a little simpler and will save a little time in authoring pages, as well as saving a small amount of file size in pages sonified with Music Object. OK, so it's not a lot, but every little bit counts, right? The other advantage to this simplified syntax is that it eliminates the plug-in specific "EMBED" tag language and helps to make magicEmbed and mo_magicEmbed more neutral to Netscape Navigator and Microsoft Internet Explorer.

Here is an example of how to change your code to make use of this simpler syntax...

BEFORE Music Object 2.7.4
myMO.magicEmbed ('<EMBED SRC="trance.rmf" WIDTH=2 HEIGHT=2 HIDDEN AUTOSTART=TRUE>');
FROM Music Object 2.7.4
myMO.magicEmbed ('SRC="trance.rmf" WIDTH=2 HEIGHT=2 HIDDEN AUTOSTART=TRUE');

  Old content that was authored before Music Object 2.7.4 should still continue to work with "<EMBED " and ">" specified.
OPTIMIZATION - General Improvements
Some optimizations were made to offset the increase in code size as a result of achieving support for WebTV. None of these code optimizations require any changes to authored content.
